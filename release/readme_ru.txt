  2.  Для чего нужна OmegaT?

OmegaT - это программа автоматизированного перевода (CAT, Computer Assisted Translation). Это бесплатная программа, а потому платить за её использование (в том числе в профессиональных целях) не придется. Кроме того, можно модифицировать и/или распространять OmegaT, при условии соблюдения пользовательской лицензии.

Основные особенности OmegaT:
  - запускается в любой операционной системе, поддерживающей Java;
  - использует технологии памяти переводов (TMX-файлы);
  - гибкое сегментирование по предложениям (по методике, схожей с SRX);
  - поиск по проекту и внешним файлам памяти переводов;
  - поиск файлов поддерживаемых форматов в любом каталоге; 
  - подбор нечётких совпадений;
  - интеллектуальная обработка проектов,в том числе со сложными иерархиями каталогов;
  - поддержка глоссариев (терминологическая справка); 
  - поддержка программ проверки орфографии с открытым исходным кодом;
  - поддержка словарей формата StarDict;
  - работа с сервисом машинного перевода Google Translate;
  - доступная и исчерпывающая документация;
  - документация и интерфейс переведены на множество языков.

OmegaT непосредственно поддерживает файлы следующих форматов:
  - неформатированный текст (plain text);
  - HTML и XHTML;
  - HTML Help Compiler;
  - OpenDocument/OpenOffice.org;
  - пакеты Java resource (.properties);
  - INI-файлы (файлы с парами «ключ=значение» любой кодировки);
  - Файлы PO;
  - файлы документации в формате DocBook;
  - Файлы Microsoft OpenXML;
  - одноязычные XLIFF-файлы стандарта OKAPI;
  - QuarkXPress CopyFlowGold;
  - Файлы субтитров (.srt);
  Файлы ResX
  - пакеты исходных данных Android;
  - LaTeX.

В OmegaT можно настроить поддержку и других форматов файлов.

OmegaT автоматически обработает даже самую сложную иерархию исходных каталогов, найдёт все поддерживаемые файлы и создаст каталог перевода с идентичной структурой, куда будут скопированы и файлы форматов, которые не поддерживаются.

Чтобы быстро ознакомиться с возможностями OmegaT, запустите программу и прочитайте краткое руководство, которое увидите в открывшемся окне.

Полное руководство пользователя поставляется вместе с программой, его можно найти в меню «Справка».

 3. Установка OmegaT

3.1 Для запуска OmegaT необходимо, чтобы на вашей системе был установлен Java Runtime Environment версии 1.5 и выше. Сейчас мы уже поставляем OmegaT вместе с JRE, чтобы избавить пользователей от необходимости выбирать, искать и устанавливать его. 

Если Java у вас уже установлен, проще всего — установить текущую версию OmegaT с помощью Java Web Start. 
Для этого, загрузите указанный ниже файл и запустите его:

   http://omegat.sourceforge.net/webstart/OmegaT.jnlp

В результате, на ваш компьютер будет установлена OmegaT и пакет прораммного обеспечения, необходимый для ее работы. В дальнейшем, наличине соединения с Интернет не обязательно.

Во время установки, в зависимости от операционной системы, могут появиться несколько предупреждений системы безопасности. Примите самоподписанный сертификат от Дидье Бриэля (Didier Briel). 
Права, которые вы даете этой версии (формулировка может быть образца «неограниченный доступ к компьютеру») ничем не отличаются от набора прав, которые есть у локально установленной версии (процедуру её установки мы рассмотрим позже), а именно: разрешается доступ к жесткому диску вашего компьютера. При повторном запуске OmegaT.jnlp, при наличии доступа в Инренет, будут загружены обновления, если они есть, и лишь затем запустится OmegaT. 

Также, установить OmegaT можно и другим способом: 

Если у вас установлены ОС MS Windows или Linux и имеется JRE нужной версии, можно загрузить «облегченную» версию OmegaT, тоесть — без JRE (установочный пакет можно отличить по маркировке «Without_JRE» после номера версии). 
Если сомневаетесь — пользуйтесь «стандартной»  версией, имеющей JRE в комплекте. Это проблем не вызовет, поскольку даже если JRE уже имеется в вашей системе, эта версия не будет с ней конфликтовать.

Если у вас установлен Linux — учтите, что OmegaT не работает с открытыми/свободными реализациями Java, которые можно найти во многих дистрибутивах Linux (к примеру, в 
Ubuntu), поскольку они либо устарели, либо не являются полными. Скачайте и установите версию Java Runtime Environment (JRE) от Sun по ссылке выше или скачайте и установите OmegaT в комплекте с JRE (архив tar.gz, помеченный «Linux»).

У пользователей Mac OS X JRE обычно уже установлена.

Если у вас установлен Linux на PowerPC, то вам придётся скачать JRE в реализации IBM,
поскольку Sun не предоставляет JRE для PPC-систем. Это можно сделать по адресу:

    http://www-128.ibm.com/developerworks/java/jdk/linux/download.html 


3.2 Установка
* Пользователи Windows: просто запустите программу установки. Если хотите, программа установки может создать ярлыки для запуска OmegaT.
* Остальные: чтобы установить OmegaT, просто создайте для неё подходящую папку (например, /usr/local/lib в Linux). Скопируйте туда архив zip или tar.gz с OmegaT и распакуйте.

3.3 Запуск OmegaT
OmegaT можно запустить несколькими способами.

* Пользователи Windows: двойным щелчком на файле OmegaT.exe. Если в файловом менеджере (Проводнике Windows) вы видите только файл OmegaT, но не OmegaT.exe, в настройках включите отображение расширений файлов.

* Двойным щелчком на файле OmegaT.jar. Этот способ работает только в случае, если в вашей системе выполнена привязка файлов с расширением .jar к Java.

* Из командной строки. Для запуска OmegaT введите:

cd <папка, в которой находится OmegaT.jar>

<путь к исполняемому файлу Java и его имя> -jar OmegaT.jar

(Исполняемый файл Java - это файл java.exe в Windows и java в Linux.
Если Java установлена на системном уровне, полный путь к нему вводить не обязательно.)

* Пользователи Windows: программа установки может создать ярлыки в главном меню, на рабочем столе и на панели быстрого запуска. Кроме того, можно вручную перетащить файл OmegaT.exe в главное меню, на рабочий стол или на панель быстрого запуска, и запускать программу оттуда. 

* Пользователи KDE в Linux: вы можете добавить OmegaT в меню следующим образом:

Центр Управления KDE - Рабочий стол - Панели - Меню - Редактировать меню К - Файл - Новый элемент/Новое подменю.

Затем, после выбора подходящего меню, добавьте новый элемент или подменю при помощи Файл - Новое подменю или Файл - Новый элемент. Введите OmegaT в качестве имени нового элемента.

В поле «Команда» найдите скрипт, запускающий OmegaT, и  выберите его. 

Щелкните на кнопке выбора значка (справа от полей Имя/Описание/Комментарий) - Прочие значки - Обзор и зайдите в подкаталог /image в каталоге OmegaT. Выберите иконку OmegaT.png.

Сохраните изменения при помощи Файл - Сохранить.

* Пользователи Linux и Gnome: вы можете добавить OmegaT в вашу панель (меню вверху экрана) следующим образом:

Щелкните правой кнопкой мыши на панели - Add New Launcher. Введите «OmegaT» в поле «Имя»; используйте кнопку обзора, чтобы ввести путь к запускающему OmegaT скрипту в поле «Команда». Выберите скрипт и подтвердите выбор нажатием кнопки OK.

 5.  Обнаружили ошибку в OmegaT? Нужна помощь?

Прежде, чем сообщать об ошибке, убедитесь, что вы внимательно прочли документацию. Возможно, это не ошибка, а особенность OmegaT. Если в лог-файле OmegaT вы видите слова вроде «Error», «Warning», «Exception» или «died unexpectedly», тогда, вероятно, вы действительно обнаружили проблему. Файл log.txt расположен в каталоге пользовательских настроек, подробнее — в руководстве.

Дальше — нужно удостовериться, что другие пользователи тоже сталкиваются с этой ошибкой и что никто не сообщил о ней до вас. Для этого просмотрите страницу сообщений об ошибках на SourceForge. Вы должны размещать сообщение об ошибке только если вы уверены, что вы первый, кто нашёл некую воспроизводимую последовательность действий, которая вызывает нечто неправильное.

Хорошее сообщение об ошибке включает три компонента:
  - Список действий, вызывающих ошибку,
  - Что ожидалось от программы, и
  - Что в итоге получилось.

Вы можете прикреплять копии файлов, фрагменты логов, скриншоты - всё, что по вашему мнению, поможет разработчикам найти и исправить ошибку.

Чтобы просмотреть архивы списка рассылки для пользователей OmegaT, зайдите на
     http://groups.yahoo.com/group/OmegaT/

Чтобы просмотреть страницу сообщений об ошибках и отправить новое сообщение, зайдите на страницу по адресу:
     http://sourceforge.net/tracker/?group_id=68187&atid=520347

Чтобы отслеживать состояние вашего сообщения об ошибке, вы, возможно, захотите зарегистрироваться на SourceFourge.
