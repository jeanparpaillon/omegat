Project files and directories

Summary of project files and directories

An OmegaT project comprises the following files and directories:

A main project directory. This directory is created when the project is created. It has the name given to it by the user.

omegat.project. When the project is created, OmegaT automatically creates this file and places it in this directory. To open a project in OmegaT, you must open its omegat.project file.

<projectname>.tmx. This is the translation memory exported by the project. OmegaT creates it automatically and places it in this directory each time you compile the project. Note that this translation memory only contains segments from the project at the time it was last compiled. It does not import segments from any translation memory files in the /tm directory. .

Directory /source. Place any files you wish to translate in this directory after creating your project and before opening it. Any files in this directory and subdirectories below it are included in the project for translation, provided their format is supported and they have the corresponding file extension.

*.txt, *.xhtml, *.html, *.sxw: source files imported into the project for translation. Files with other extensions can also be imported if their extensions are mapped to the default extensions.

Files in /source which are not imported into the project are transferred unchanged to /target when you compile the project. This enables complete web sites in /source to be mirrored in /target, together with their image files in formats such as GIF or JPEG.

Directory /glossary. Place a glossary which you wish to use in this directory after creating the project.

*.tab: glossary files.

Directory /omegat. OmegaT automatically places the following files in this directory at creation:

file_extension_mapping: OmegaT detects file types on the basis of their extensions; StarOffice files, for example, must have the extension .sxw in order to be recognized. You can force OmegaT to recognize files with different extensions (but not formats!) by editing this file.

project_save.tmx: this file is created when the project is saved for the first time. It is in TMX format and is the working translation memory for the project.

word_counts: this file contains two word counts of the complete project: one including and one excluding 100% matches.

Directory /target. OmegaT places the translated files into this directory at compilation.

*.txt, *.xhtml, *.html, *.sxw: translated files
Non-translated files transferred from /source at compilation

Directory /tm. Place any translation memories you wish to use in this directory after creating/before loading the project. Note: regardless of whether you use previous memories or not, OmegaT creates a working translation memory for the project, project_save.tmx, in /omegat when the project is saved, and also places an export translation memory in the root directory when the project is compiled.

*.tmx: translation memory files created during previous projects. *.tmx files must be in TMX Level 1 format.

Changing the location of project files and directories

The project directories are created when you create a project. By default, they are all grouped under the project root directory, but you can choose alternative locations for them by clicking on the Browse button. You can therefore keep translation memories in a particular directories, for example, and select the desired directory when you create each new project rather than having to copy the translation memory file into the newly created /tm directory.

To change the location of directories retrospectively, i.e. those of an existing project, delete the directory concerned. (Do not do this whilst the project is open.) When the project is next opened, OmegaT detects that some directories are missing, and presents you with the directory selection dialog in which you can select the new locations.
