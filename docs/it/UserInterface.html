User Interface

The graphical user interface of OmegaT consists mainly of an Editor window and a Match and Glossary Viewer window.

The Editor displays the text currently in the process of translation. The state of each segment is clearly indicated in the editor pane. Untranslated segments are displayed in the source language and translated segments are displayed in the target language. The only exception to this is the active segment where translation and the revision take place. It has two features: the segment is displayed in an editing field and bracketed by <segment number> and <end segment> tags; and the source language text is displayed above the editing field in highlighted bold characters for easy reference;

The Match and Glossary Viewer is divided into two panes. The Match pane displays fuzzy matches (previously translated segments with a certain degree of similarity to the active segment). The Glossary pane displays information from the glossary file, if one exists. This window is automatically displayed when OmegaT is started. It can be opened and closed with the File > Match/Glossary Viewer... menu item.

There is also a Project Files window, which contains a list of files for translation in the current project, together with the number of segments in each file. This automatically opens when a project is first opened. Access to it is available through the File > Project Files... menu item when it is closed. Select a file for translation by clicking on its name in the window.

All of OmegaT's functionality is available through the menu bar at the top of the Editor window. Most functions are also available via keyboard shortcuts.

A status bar is visible just beneath the menubar once a translation project has been opened for work to be done. This gives the user feedback messages about specific operations that are in progress or errors that have been encountered.
