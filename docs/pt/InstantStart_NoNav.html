Este guia de início rápido aborda apenas os pontos básicos. O ideal é que os usuários leiam o Manual do Usuário do OmegaT para se familiarizarem com todos os recursos que o programa oferece.

Em resumo, como o OmegaT funciona

OmegaT é composto de duas janelas que podem ser movidas de forma independente na tela. A janela da esquerda é a janela de edição onde você digitará sua tradução. A janela da direita é do Visualizador de Correspondência/Glossário, onde as correspondências imperfeitas (caixa superior) e as correspondências do glossário (caixa inferior) serão exibidas.

OmegaT extrai os segmentos do documento fonte, exibe-os na tela de edição para serem traduzidos e acrescenta sua tradução à memória de tradução. Quando a tradução estiver terminada (ou a qualquer momento que o usuário solicitar), o OmegaT cria um documento destino, reconstruindo o documento fonte com os segmentos da memória de tradução.

OmegaT organiza seu trabalho de tradução em projetos de tradução. Para cada projeto, o OmegaT cria um conjunto de pastas nas quais você deve copiar os documentos fonte e eventuais glossários e memórias de tradução existentes. OmegaT também cria uma pasta que conterá os documentos traduzidos.

Como traduzir usando o OmegaT

1. Criar e abrir um novo projeto

Selecione o menu Projeto -> Novo....

Acesse a pasta onde gostaria de gravar os arquivos do projeto e digite um nome para o projeto de tradução. Este nome será também o nome da pasta do projeto onde todos os arquivos a ele relacionados serão armazenados.

OmegaT mostrará uma mensagem solicitando ao usuário para mudar ou aceitar as pastas do projeto a serem criadas. A tela é parecida com esta imagem:

Você pode simplesmente aceitar a localização sugerida das subpastas (diretórios), mas deverá verificar se os códigos do idioma fonte e idioma destino estão corretos. Selecione o idioma ou códigos de idioma e país na lista ou então digite-os manualmente. Observe onde está localizada a subpasta "destino", pois os documentos destino serão exportados para ela.

Se você quiser que o OmegaT faça a segmentação por frase em vez de por "parágrafo", recomendamos verificar novamente as Regras de Segmentação.

Após clicar em OK para aceitar a configuração do projeto, o OmegaT exibe uma mensagem para você importar os documentos fonte. É possível importar arquivos individuais ou então importar toda uma árvore de diretórios (com todos os arquivos em todas as subpastas). Se importar por engano arquivos errados ou arquivos em excesso, basta retirá-los da pasta "destino" do seu projeto OmegaT.

Lembre-se, o OmegaT somente aceita para tradução arquivos do OpenDocument, documentos do OpenOffice.org, arquivos em HTML, em texto e certos arquivos Java. Outros tipos de arquivos serão ignorados pelo OmegaT.

Verifique se Exibir -> Arquivos do projeto... está ativada e depois selecione Projeto -> Recarregar para ver uma lista de todos os arquivos a serem traduzidos. O OmegaT está ajustado para carregar o primeiro arquivo na lista do projeto.

2. Traduzir os segmentos um por um

Para o segmento aberto, o OmegaT destaca o texto fonte em verde e exibe uma cópia deste texto entre dois rótulos <segment 0001> e <end segment>. Digite a tradução entre os dois rótulos, sobrescrevendo o texto fonte.

Pressione "ENTER" para seguir ao próximo segmento.

Ao pressionar "ENTER", o OmegaT processa diversas ações que não visíveis ao usuário: acrescenta o segmento traduzido à memória de tradução e busca na memória e no glossário correspondências para o segmento não traduzido.

Se o próximo segmento tiver uma correspondência imperfeita na memória de tradução, ela será exibida na caixa superior do Visualizador de Correspondência/Glossário. Se o segmento tiver algum termo correspondente no glossário, ele será mostrado na caixa inferior do Visualizador de Correspondência/Glossário.

Se houver uma correspondência imperfeita, ela pode ser inserida na janela de edição da seguinte forma:

  • Selecione a correspondência imperfeita usando Ctrl+1, Ctrl+2, etc.
  • Pressione Ctrl+I para inserir o texto onde o cursor está posicionado, ou...
  • Pressione Ctrl+R para sobrescrever todo o segmento aberto com o texto da memória.

Se quiser que o OmegaT não copie o texto fonte no campo destino, ou se preferir que o texto da memória, com uma porcentagem alta de correspondência seja automaticamente inserido no campo destino, defina essas opções no menu Opções -> Fluxo de trabalho....

3. Criar documentos destino

Quando a tradução estiver terminada (ou a qualquer momento que o usuário solicitar), o OmegaT pode criar o documento destino, reconstruindo o documento fonte com os segmentos da memória de tradução.

Para criar os documentos destino, selecione o menu Projeto -> Criar documento destino. O OmegaT criará as versões destino de todos os documentos válidos no diretório "fonte" do projeto, não importando quanto do documento já foi traduzido.

Para ver a qualquer momento o documento destino, basta acessar 'Criar documento destino'. Você pode alterar as traduções existentes no projeto de tradução e depois recriar os documentos destino com a freqüência que quiser.

Os documentos traduzidos agora estão disponíveis no diretório "destino" do projeto.

Trabalhar com texto com rótulos

Os arquivos do OpenDocument, OpenOffice.org ou em HTML geralmente apresentam frases onde determinadas palavras possuem um formato especial. Por exemplo, uma frase pode estar em itálico, porém uma palavra no meio dela pode estar em negrito.

O OmegaT marca essas modificações de formato com rótulos especiais que devem ser mantidos na versão traduzida do texto. Consulte o Manual do Usuário OmegaT para informações de edição avançada de texto com rótulos.

Um exemplo...

Nestes exemplos destacamos os rótulos em vermelho para facilitar a visualização, mas no OmegaT eles não estarão destacados.

Aqui está um exemplo de um segmento em HTML:

<p>Para selecionar uma fonte de visualização diferente, clique na caixa de diálogo <b>Fonte de visualização</b>. Para abrir esta caixa, clique no menu <i>Ajustes</i> &gt; <i>Fonte de visualização...</i>. O tipo e tamanho da fonte podem ser modificados nesta caixa.</p>

Aqui está como ele será exibido no OmegaT, destacado em vermelho:

Para selecionar uma fonte de visualização diferente, clique na caixa de diálogo <b0>Fonte de visualização</b0>. Para abrir esta caixa, clique no menu <i0>Ajustes</i0> > <i1>Fonte de visualização...</i1>. O tipo e tamanho da fonte podem ser modificados nesta caixa.

Aqui está como você traduziria o texto (o exemplo está em africâner):

'n Mens kan 'n ander vertoonfont kies met die <b0>Vertoonfont</b0>-dialoogkassie. Kies <i0>Opstelling</i0> > <i1>Vertoonfont...</i1> op die kieslys. Die lettertipe én die lettergrootte kan met dié dialoogkassie verander word.

Quando o OmegaT cria o documento destino, o texto em HTML terá a seguinte marcação:

<p>'n Mens kan 'n ander vertoonfont kies met die <b>Vertoonfont</b>-dialoogkassie. Kies <i>Opstelling</i> &gt; <i>Vertoonfont...</i> op die kieslys. Die lettertipe én die lettergrootte kan met dié dialoogkassie verander word.</p>

O OmegaT não detecta automaticamente os erros em uma tradução com rótulos. Antes de entregar o documento traduzido para seu cliente, verifique se há erros de rótulos; para isto, selecione o menu Ferramentas -> Validar rótulos. Recomenda-se que você abra o documento no visualizador ou editor ao qual ele está associado para verificação final da formatação.
