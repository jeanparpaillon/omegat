Erlernen Sie den Umgang mit OmegaT in 5 Minuten! Kurzeinführung

Diese Kurzeinführung behandelt nur die absoluten Grundlagen. Im Idealfall sollten Anwender das Benutzerhandbuch lesen, um sich mit allem, was OmegaT zu bieten hat, vertraut zu machen. Die hierin beschriebenen Tastenkombinationen folgen dem Schema "Strg+Taste". Für Mac-Anwender gilt stattdessen "cmd+Taste". Die "cmd" Taste ist mit "Befehl" gekennzeichnet oder zeigt einen Apfel.

Kurz und knapp: Wie OmegaT funktioniert

Die OmegaT Oberfläche ist ein in drei Fenster aufgeteiltes Feld. Jedes dieser Fenster kann mithilfe der Maus in der Größe verändert werden. Auf der linken Seite befindet sich der "Editor", in den die Übersetzung eingegeben wird. Auf der rechten Seite oben befindet sich das "Match Viewer"-Fenster, in welchem Treffer aus der Translation Memory (TM) angezeigt werden. Darunter ist das "Glossary Viewer"-Fenster, das Übereinstimmungen mit dem Glossar anzeigt.

Das Editor-Fenster enthält den Ausgangstext in satzweiser Segmentierung (oder nach Absätzen). Diese Segmente werden nacheinander übersetzt. Ihre Übersetzungen werden dann Segment für Segment in einer "Translation Memory" (TM) gespeichert. Nachdem alle Segmente übersetzt wurden (oder vorher, wenn gewünscht), wird OmegaT die übersetzten Dokumente mithilfe der TM im Ordner "Target" erstellen.

In OmegaT werden Übersetzungen in so genannten Übersetzungsprojekten angelegt. Für jedes Projekt erstellt OmegaT einen Satz an Ordnern. In diesen Ordnern werden die zu übersetzenden Ausgangsdokumente, etwaige von Ihnen verwendete Glossare und TM's, die Sie evtl. einsetzen möchten, abgelegt. OmegaT erstellt auch einen Ordner namens "Target", in dem die fertig übersetzten Dokumente abgelegt werden.

Übersetzen mit OmegaT

Erstellen und Öffnen eines neuen Projektes

Wählen Sie im Menü Projekt → Neu... aus.

Gehen Sie zum Ordner, in dem Sie die Projektdateien speichern möchten und geben Sie einen Namen für das Übersetzungsprojekt an. Dieser Name wird für den Hauptordner des Projekts benutzt; alle Projektdateien werden in diesem oder seinen Unterordnern gespeichert.

OmegaT wird Sie auffordern, die zu erstellenden Projektordner zu bestätigen oder zu verändern, und dabei den unten stehenden Dialog verwenden:

Sie können die Lage der Unterordner einfach übernehmen, vergewissern Sie sich aber zuerst, dass die Codes für Ausgangs- und Zielsprache korrekt sind. Wählen Sie den Sprachcode (2 Buchstaben) oder den Sprach- und Ländercode (2 + 2 Buchstaben) aus der Dropdown-Liste, oder tippen Sie einen manuell ein (halten Sie das 2 Buchstaben Muster ein). Merken Sie sich, wo sich der Unterordner "Target" befindet (in dem sich Ihre übersetzten Dateien befinden werden). Wenn Sie möchten, dass OmegaT satz- anstatt abschnittsweise segmentiert, ist es sinnvoll, die Segmentierungsregeln erneut zu überprüfen.

Nachdem Sie auf OK geklickt haben, um die Projekteinstellungen zu akzeptieren, wird OmegaT Sie auffordern, die Ausgangsdokumente auszuwählen, um diese zu importieren. Sie können einzelne Dateien importieren, oder Sie können komplette Verzeichnisbäume (mit allen Dateien in allen Unterverzeichnissen) importieren. Sollten Sie aus Versehen die falschen oder zu viele Dokumente importiert haben, können Sie diese einfach aus dem Verzeichnis "Source" Ihres OmegaT-Projektes löschen (benutzen Sie dafür z.B. den Dateimanager Ihres Systems).

Die Liste der zu übersetzenden Dateien kann im Fenster Projektdateien überprüft werden (Menü: Projekt → Projektdateien..., falls das Fenster sich nicht automatisch öffnet). Wenn Sie den Inhalt des Ordners "Source" ändern mussten, sollten Sie nicht vergessen, das Projekt zuerst neu zu laden (Menü: Projekt → Projekt neu laden). Als Grundeinstellung öffnet OmegaT die erste Datei in der Projektliste. Bedenken Sie: OmegaT kann nur Dateien in den unten aufgelisteten Formaten übersetzen, vorausgesetzt, sie entsprechen den Mustern, die in Dateifiltern definiert sind. Alle anderen Dateien werden ignoriert.

  • OpenDocument/OpenOffice.org
  • Reiner Text
  • Einsprachige .po
  • Bundle.properties Java
  • XHTML, HTML
  • HTML Hilfe Erstellung
  • INI ('Schlüssel=Wert' Format)
  • DocBook
  • Microsoft Open XML
  • Okapi einsprachige XLIFF

Übersetzen Sie ein Segment nach dem anderen

Nachdem Sie das Projekt und die zu übersetzenden Dateien festgelegt haben, öffnet OmegaT die erste Ausgangsdatei im Editor. Das erste Segment wird grün unterlegt dargestellt; eine Kopie des Ausgangstextes erscheint im "Zielfeld" darunter. (An diesem Punkt wird der ganze Text außerhalb des Zielfeldes geschützt und kann nicht überschrieben werden.) Die Übersetzung wird zwischen den Tags <Segment 0001> und <Segmentende> eingetippt und dabei wird der Ausgangstext überschrieben. "ENTER" drücken, um zum nächsten Segment zu gelangen. Hinweis:

  • Um zu einem anderen Teil des Textes, entweder oben oder unten, zu gelangen, klicken Sie doppelt auf den Satz oder Paragrafen, den Sie öffnen möchten.
  • Wenn Sie es vorziehen, in ein leeres Zielfeld hinein zu übersetzen, können Sie das in Optionen → Editierverhalten... einstellen.

Wenn Sie "ENTER" drücken, geschehen einige Dinge unsichtbar: OmegaT fügt das Segmentpaar (das Ausgangssegment und seine Übersetzung) zur TM hinzu und übersetzt automatisch irgend welche identischen Segmente, die es in den anderen Projektdateien findet. Außerdem werden TM und Glossare auf Treffer für das nächste, noch nicht übersetzte Segment durchsucht.

Verwendung von Übereinstimmungen aus TM und Glossar

Übereinstimmungen mit Translation Memory (TM)

Wenn OmegaT sog. Fuzzy Matches (30 %ige oder höhere Gleichheit) für das nächste Segment findet, werden diese im Match Viewer angezeigt. Der erste Treffer im Match Viewer ist standardmäßig bereits ausgewählt. Um Übereinstimmungen aus diesem Fenster in das Übersetzungsfeld einzufügen, können Sie Tastenkombinationen verwenden:

  • Drücken Sie Strg+I um den vorausgewählten Treffer an der Cursorposition einzufügen, oder...
  • Drücken Sie Strg+R um das gesamte Segment mit dem vorausgewählten Treffer zu überschreiben.

Wenn mehrere Treffer vorliegen und Sie einen anderen als den vorausgewählten verwenden möchten:

  • Wählen Sie den Treffer aus dem Match Viewer aus, den Sie einfügen möchten: Strg+2 für den zweiten angezeigten Treffer, Strg+3 für den dritten Treffer, usw.
  • Drücken Sie dann Strg+I oder Strg+R wie oben beschrieben.

(Sie können OmegaT auch so einstellen, dass der erste Treffer, der oberhalb einer gewissen Schwelle liegt, automatisch in das Übersetzungsfeld eingefügt wird, sobald Sie dies öffnen. Diese Einstellung finden Sie unter Optionen → Editierverhalten...)

Übereinstimmungen mit dem Glossar

100% Treffer aus dem Glossar werden als Referenz im Glossary Viewer dargestellt. Sie können nicht mittels Tastenkombination übertragen werden.

Anwendung von Suche

OmegaT hat ausgefeilte Suchfunktionen. Die Suchfunktion wird mit Strg+F aufgerufen, dann das zu suchende Wort oder den Satz in das Feld "Suche nach" eingeben. Wahlweise kann ein Wort oder ein Satz im Editierfeld markiert (Zieltext des aktiven Segments) und Strg+F gedrückt werden. Das Wort oder der Satz wird in diesem Fall automatisch in das Feld "Suche nach" eingegeben.

Erstellen der übersetzten Dokumente

Nach Übersetzung aller Segmente (oder, auf Wunsch, auch schon früher) aktualisiert OmegaT das/die Zieldokument(e) mithilfe der Übersetzungen, die im TM gespeichert sind. Dazu wählen Sie aus dem Menü Projekt → Zieldokumente erstellen. OmegaT erstellt dann Übersetzungen aller Dokumente, die sich im Ordner "Source" befinden und mittels OmegaT übersetzt werden können, ganz gleich ob sie bereits vollständig übersetzt sind oder nicht. Die ganz oder teilweise übersetzten Dokumente werden dann im Ordner "Target" abgelegt. Um die Übersetzung abzuschließen, öffnen Sie die Zieldateien in den jeweiligen Anwendungen (Browser, Textverarbeitungsprogramm...), um den Inhalt und die Formatierungen zu überprüfen. Sie können dann etwaige Änderungen in OmegaT anbringen. Vergessen Sie nicht, die Zieldokumente anschließend wieder neu zu erstellen.

Wie Text mit "tags" gehandhabt wird

OmegaT stellt mit Markierung durch sog. "Tags" sicher, dass die Formatierung der Ausgangsdokumente (Fettschrift, Kursivschrift, usw.) nicht verloren geht. OmegaT "Tags" sind aus einem oder mehreren Buchstaben, gefolgt von einer oder mehreren Zahlen zusammengesetzt, wobei das "/" Zeichen zum Schließen dient (zum Beispiel: <f0>, </f0> <br>, </s2> etc.). Behandeln Sie diese Tags mit Vorsicht und stellen Sie sicher, dass sie auch in den Zielsegmenten korrekt enthalten sind (SieheTag Operationen).

Verifizierung von Tags

OmegaT erkennt fehlerhafte Tags im übersetzten Dokument nicht automatisch. Vor Abgabe des übersetzten Dokuments sollten Sie prüfen, ob Tag-Fehler vorliegen.

Wählen Sie aus dem Menü Extras → Tags prüfen aus. Bei vorliegenden Fehlern erscheint eine Tabelle, die Segmente zeigt, bei denen die Tags in Ausgangs- und Zieltext nicht übereinstimmen. Klicken Sie auf die Segmentnummer. Im Editor-Fenster wird dann automatisch das fragwürdige Segment geöffnet, und Sie können die Tags korrigieren.

In manchen Fällen können Tag-Fehler das Öffnen der übersetzten Dokumente unmöglich machen. Sie sollten daher vor dem Erstellen der Zieldokumente überprüfen, ob Sie alle Tag-Fehler korrigiert haben. Schließlich sollte auch immer die Formatierung überprüft werden, indem das übersetzte Dokumente in der jeweiligen Anwendung geöffnet wird.
