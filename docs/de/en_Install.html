Installation und Konfiguration

Installation

Die Installation besteht aus drei Schritten. Erstens muss J2RE installiert werden (sofern es noch nicht installiert ist). Zweitens muss OmegaT installiert werden. Drittens muss die Konfiguration erfolgen. Es ist unerheblich in welcher Reihenfolge J2RE und OmegaT installiert werden. Die Konfiguration ist nicht unbedingt notwendig, aber das Aufrufen der Anwendung wird dadurch benutzerfreundlicher.

Installation des Java Runtime Environments

Das Java Runtime Environment (J2RE) erlaubt es, Java-Programme auf dem eigenen Computer auszuführen. Es steht sowohl als Einzelprodukt als auch als Teil des Java-Entwicklerkits zur Verfügung.

Java gibt es auch als Plugin für Webbrowser. Mit dieser Art von Java ist es nicht möglich mit OmegaT zu arbeiten.

JavaScript ist ein anderes Produkt und hat technisch absolut nichts mit Java zu tun.

Wenn man mit OmegaT arbeiten möchte, muss die J2RE auf dem Computer installiert sein. OmegaT funktioniert nicht ohne J2RE. Versuchen Sie nicht, OmegaT ohne J2RE zu starten. Es wäre reine Zeitvergeudung!

Wenn man nicht sicher ist, ob J2RE auf dem Computer installiert ist, reicht es aus OmegaT zu installieren und zu sehen, ob es funktioniert (die Installation von OmegaT ist nachfolgend beschrieben). Wenn nichts geschieht oder Fehlermeldungen angezeigt werden ist es sehr wahrscheinlich, dass das Java Runtime Environment nicht installiert ist. Sollte dies der Fall sein, muss man es sich besorgen und installieren.

Windows Anwender: um zu kontrollieren, welche Version des Java Runtime Environments installiert ist (sofern sie installiert ist) indem man folgende Tasten drückt/Felder aktiviert: Start-Taste/Windows-Taste >Einstellungen > Systemsteuerung. Wenn J2RE installiert ist, ist das Java-Symbol für das “Java(TM)2 Runtime Environment, Standardversion 1.3.0_01” oder ähnlich vorhanden.

Linux Anwender: einfach Java -version in die Kommandozeile eingeben. Die installierte Version wird als Antwort angezeigt (sofern diese natürlich auf dem System installiert ist).

J2RE kann man bei nachfolgenden Links herunterladen. Es ist gratis (siehe Lizenzbedingungen), aber die Namenskonventionen können das Auffinden der richtigen Version etwas schwierig gestalten. Man benötigt die richtige Version für das eigene Betriebssystem (Linux, Windows, usw.). Die Dateigröße des J2RE ist ca. 14 MB.

(Hinweis: um seine Anwender durcheinander zu bringen ändert Sun regelmäßig die Namen seiner Produkte. Also sind alle Versionen von JRE über 1 als “Java 2” bekannt, deshalb heißt das Java Runtime Environment jetzt J2RE. Das Java Runtime Environment ist auch als Java Virtual Machine oder JVM und bei Macintosh als MRJ bekannt. Das Entwicklerkit ist als Java Developer's Kit (JDK), Software Developer's Kit (SDK), und nun auch als J2SE, J2EE und J2ME, je nach Version, bekannt. Zumindest meine ich, dass es so ist. Aber ich bin bei denen dabei, denen das ganze nicht so klar ist.)

OmegaT 1.4.2 müsste mit jeder Version des J2RE ab 1.2 oder höher laufen. Wir empfehlen auf jeden Fall die neueste Version von J2RE, da ältere Java-Versionen bestimmte Eigenschaften eventuell nicht unterstützen.

Als Alternative zu J2RE kann man auch das Entwicklerkit installieren. Das Entwicklerkit beinhaltet eine eigene Version von J2RE und erlaubt es, OmegaT zu verändern.

Windows und Linux Anwender können das Java Runtime Environment hier herunterladen:

Sun (alle Benutzeroberflächen)

Blackdown (Linux)

Bei Macintosh OS X Anwendern sollte das Java Runtime Environment bereits installiert sein.

Es ist bekannt, dass frühere Versionen von OmegaT auf dem MRJ für Macintosh OS 9 laufen. Die Version 1.4.2 sollte auf dieser Benutzeroberfläche funktionieren, was allerdings noch nicht getestet wurde.

Während der J2RE-Installation sind die dazugehörigen Anweisungen zu beachten.

Installation von OmegaT

Einfach den Inhalt des Archivs in das gewünschte Verzeichnis extrahieren. Das Archiv kann in *.zip oder *.tar.gzip Form sein. Anwender von Linux und Mac (Stuffit) dürften keinerlei Probleme mit beidem haben; bei Windows Anwendern ist es möglich, dass sie kein Tool für letztere haben.

Aufruf der Anwendung

Windows und Macintosh OS X

Auf die Datei OmegaT.jar im Windows-Explorer klicken.

Linux

Java -jar OmegaT.jar über die Kommandozeile ausführen. Die Schreibweise von OmegaT ist genau zu beachten - die Großbuchstaben sind wichtig.

Wenn Java nicht dafür konfiguriert wurde, den Ort der OmegaT-Dateien zu finden, müssen zuerst die Verzeichnisse auf jene umgestellt werden, die die Datei OmegaT.jar enthalten.

Wenn man die Java-Pfadvariable nicht eingestellt hat, muss man den kompletten Pfad angeben. Wenn keiner von den beiden oben beschriebenen Schritten ausgeführt wurde, ist es möglich, dass die Routine zum Aufruf der Software nach einem ähnlichen Pfad wie den folgenden sucht:

cd /home/user/OmegaT
/home/user/j2re1.4.2/bin/java -jar OmegaT

Konfiguration (nur für Linux)

Wenn man nicht sehr gut mit Linux umgehen kann, dann macht folgender Schritt das Aufrufen von OmegaT anwenderfreundlicher:

Erstellen eines Shell-Scripts mit Hilfe einer Textdatei folgenden Inhalts:

#!/bin/bash
cd <OmegaT>
<java>java -jar OmegaT.jar

wobei <OmegaT> das Verzeichnis ist, in dem sich OmegaT.jar befindet und <java> das Verzeichnis mit der J2RE Binärdatei ist (die Datei mit Namen java). Wenn der Java-Pfad als variabel eingestellt ist, muss dieses Verzeichnis nicht angegeben werden und java -jar OmegaT.jar ist ausreichend. Die Angabe des Verzeichnisses ist notwendig, wenn der Java-Pfad nicht als variabel eingestellt wurde und wenn auf dem System mehr als eine Java-Version installiert sind und sicherzustellen ist, dass OmegaT mit der richtigen Version aufgerufen wird.

Die Textdatei als OmegaT speichern und die Rechte auf ausführbar setzen (z.B. bei Konqueror, Kontextmenü > Eigenschaften > Rechte > Ausführen). Man kann OmegaT nun ausführen, indem man auf der Kommandozeile einfach OmegaTeingibt. Wenn man ein Symbol auf dem Desktop anzeigen möchte, so ist dies auf Ausführung der Shell-Script-Datei einzustellen.
