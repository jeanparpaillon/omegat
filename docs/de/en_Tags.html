Editieren von formatiertem Text

Mit Tags arbeiten

OmegaT zeigt die eingebetteten Formatierungsinformationen von unterstützten Formaten als Tags an (im Moment sind das HTML,XHTML und OpenOffice.org/StarOffice). Die Tags, die während der Arbeit mit OmegaT zu sehen sind, sind nicht dieselben, die die Originalanwendungen verwenden; OmegaT konvertiert sie intern und re-konvertiert sie beim Kompilieren.

Die Tags müssen unbedingt gleich belassen werden. Sollten sie irrtümlich verändert werden, kann die Formatierung der Übersetzung verloren gehen. Im schlimmsten Fall kann man die übersetzte Datei nicht öffnen: dies passiert insbesondere mit OpenOffice.org Dateien.

Auch wenn man selbst keinerlei Formatierung ausführen kann, hat man bis zu einem bestimmten Punkt die Möglichkeit, die existierende Formatierung zu beeinflussen.

Manipulieren von Tags bei OpenOffice.org-Dateien

Die Tags von OpenOffice.org müssen unverändert erhalten bleiben. Man darf diese Tags nicht löschen, vervielfältigen oder deren Reihenfolge verändern. Man kann dagegen:

  • die Tags an eine andere Stelle des Segments bewegen; sie müssen auf jeden Fall in derselben Reihenfolge bleiben;

  • Tags löschen, indem man sie hintereinander stellt. Wenn man z.B. diesen Text

    Dieser Text ist <f0>kursiv</f0>
    in
    Dieser Text ist <f0></f0>kursiv

    verändert, wurde die Formatierung für Kursiv entfernt.

Manipulieren von Tags in HTML-Dateien

  • Der Beginn der Formatierung ist von einem Eröffnungstag gekennzeichnet, der folgende Form hat: <xx>; das Ende der Formatierung weist einen entsprechenden Abschlusstag auf: </xx>. Irgendwelche Änderungen bei der Formatierung müssen an beiden Tags des Paars ausgeführt werden.

  • Man kann Tags löschen . Wenn man z.B. die Kursivschrift von nachfolgenden Tag entfernen will:

Dieser Text ist <f0>kursiv</f0>

löscht man einfach die Tags <f0> und </f0>.

  • Man kann Tags duplizieren . Zum Beispiel:

Dieser Text ist <f0>kursiv</f0>, und so ist <f0>das</f0>

- die "kursiv"- Formatierung ist damit auch bei dem Wort “das” gültig.

  • Man kann die Reihenfolge von Tags ändern. Zum Beispiel:

Dieser Text ist <f0>kursiv</f0>, dieser Text ist <f1>unterstrichen</f1>

kann geändert werden in

Dieser Text ist <f1>unterstrichen</f1>, dieser Text ist <f0>kursiv</f0>

  • Man kann Tags ineinander verschachteln . Zum Beispiel:

Dieser Text ist <f0>kursiv</f0>, dieser Text ist <f1>unterstrichen</f1>

kann geändert werden in

Dieser Text ist <f0>kursiv und <f1>dieser Text ist auch unterstrichen</f1></f0>

  • Die Tags dürfen sich nicht überschneiden. Zum Beispiel:

Dieser Text ist <f0>kursiv, dieser Text ist auch <f1>unterstrichen</f0>, dieser Text ist unterstrichen, aber nicht kursiv</f1>

ist nicht erlaubt und macht die Datei kaputt, wenn man versucht die Tags so platzieren. Wenn man einen Eröffnungstag und dann danach einen zweiten Eröffnungstag einfügt, muss der Abschlusstag zu dem zweiten Eröffnungstag vor dem Abschlusstag des ersten eingefügt werden.

Prüfen der Tags

Die Funktion zum Prüfen der Tags (Extras > Tags prüfen) sucht Tags, die (absichtlich oder unbeabsichtigt) geändert wurden und zeigt die betroffenen Segmente an.
