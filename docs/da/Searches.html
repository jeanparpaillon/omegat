Searches: Keyword and Exact

  1. Summary
  2. Exact Searches
  3. Keyword Searches

Summary

OmegaT provides a powerful range of search functions.

Open a search window

Press <Ctrl+F> or select Search Project... from the Edit menu.

To keep track of previous searches calling a search window will always create a new window.

Search a string

Select a word or phrase in the editing field (target text of the active segment) and open a search window. The word or phrase is automatically entered into the Search for box.

Alternatively, enter a word or phrase to search for in the Search for text entry area of an opened search window.

Options

Select the type of search using the radio buttons (exact or keyword search).

Both search types give you access to search TMs, only exact searches give you access to search files (recursively or not).

All search types given are case-insensitive. That is, a search for "abc" will find instances of "abc", "ABC", "Abc", etc.

Exact Searches

An exact search looks for a character string. Therefore, if several words are entered, they are only found when they occur in exactly the same sequence. The searched string can also be an incomplete word.

Wildcard characters

Wildcard characters (glob pattern characters) such as * and?. can be used in exact searches:

  • An asterisk * matches any character or number of characters. For instance, "b*d" matches instances of "bad", "bid", "bold" and "band";
  • A question mark ? matches any single character. "b?d" will match instances of "bad" and "bid".

Exact search location

When Search TMs" is on (default setting):
OmegaT searches for the string in the source and target segments of the current project, in the translation memories (files in /tm) as well as in the project translation memory (/omegat/project_save.tmx).
When Search TMs is off:
OmegaT searches for the string exclusively in the source and target segments of the current project
When Search Files is on:
OmegaT searches the string in any file in a format supported by OmegaT (e.g. *.txt, *.html, *.xhtml, *.odt, *.sxw, *.properties), located in the set directory.
When both Search Files and Recursive search are on:
OmegaT searches the string in any file in a format supported by OmegaT (e.g. *.txt, *.html, *.xhtml, *.odt, *.sxw, *.properties), located in the set directory and any subdirectory existing.

Keyword Searches

Select Keyword search and enter any number of individual words, in any order, and OmegaT displays a list of all segments in which all the words occur.

Only whole words, not parts of words or incomplete words, can be found.

Keyword search location

When Search TMs" is on (default setting):
OmegaT searches for the keywords in the source and target segments of the current project, in the translation memories (files in /tm) as well as in the project translation memory (/omegat/project_save.tmx).
When Search TMs is off:
OmegaT searches for the keywords exclusively in the source and target segments of the current project

Search files cannot be activated with Keyword searches.
