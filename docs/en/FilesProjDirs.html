Project Files and Directories

An OmegaT project is comprised of a number of files and directories.

Files

  • log.txt: an automatically created log file used to check for errors. Users can check this file and report errors and bugs to the users group or development team (see Important Notes also).

  • omegat.project. OmegaT automatically creates this file when the project is created. It is located in the main project directory. This file can be edited in any text editor to modify some project parameters. Only modify this if you know what you are doing.

  • <projectname>.tmx. This is the translation memory exported by the project. It is saved in the main project directory and is automatically updated each time the project is compiled. This translation memory contains all the segments from the last time the project was compiled. Note that it does not import segments from any translation memory files contained in the /tm directory.

  • project_save.tmx: this is a TMX file that is created in /omegat when the project is saved for the first time. This serves as the working translation memory for the project. It is possible to merge pre-existing TMs (of the same type) into this one.

  • project_save.tmx.bak: this is a TMX file that is created in /omegat when a project is first reopened. It serves as a backup of project_save.tmx: and is updated each time you reopen a project, before any possible change has taken place to the translation memory for the project.

  • *.tmx in /tm: translation memory exchange files created previously. These files must be in TMX Level 1 (version 1.1) format.

  • word_counts: contains two word counts of the complete project: one count including and one count excluding 100% matches (this function is currently disabled; it will be re-enabled in a newer versions of OmegaT)

  • source document files in /source to be imported into the project for translation. Files with other extensions can also be imported if their extensions are mapped to default extensions. See Source Document Files and File Filters for information on file types and mappings.

  • other (unsupported) files in /source. These are copied unmodified to /target when a project is compiled. This enables, for instance, the inclusion of files for a complete web site in /source to be mirrored in /target, together with any image files (e.g. GIF, JPEG, or PNG images).

  • translated document files (of the same type as the associated source document files in /source), along with other non-translatable files copied from /source at compilation.

  • glossary files (i.e. *.tab, *.utf8 file types) in /glossary.

Directories

  • A main project directory. This directory is created when the project is created. It has the name given to it by the user.

  • /glossary. Contains glossary files to be used, if any. The user must place glossary files in this directory before opening a project.

  • /omegat. Contains the files project_save.tmx, project_save.tmx.bak, and word_counts.

  • /source. Any files to be translated are placed into this directory before opening a project. Any files in this directory and its subdirectories are included in the project for translation, provided that their formats are supported and they have the corresponding file extensions.

  • /target. Translated (and non-translatable) files are located in this directory after compilation.

  • /tm. Any translation memories to be used are placed in this directory before loading a project. Note: regardless of whether you use previously made translation memories or not, OmegaT will create project_save.tmx, the working translation memory for the project (located in /omegat) when the project is saved. It also places an exportable translation memory in the main project directory after the project is compiled.

Changing the location of project files and directories

The project directories are created along with the project. By default, they are all grouped under the main project directory. Alternate locations for the directories can be chosen by clicking on the Browse... button when creating a project. It is then possible to keep translation memories in a particular directory (e.g. as a TM repository) and select this directory when you create a new project rather than having to copy translation memories into a new project's /tm directory.

To change the location of directories after a project has been created it is necessary to alter a project's accepted structure from its default settings. First, ensure that a project is not open. Then move or copy one or more directories or their files to a new location. If only copying is done, delete the original directory or directories where it was done from. Upon re-opening the project, OmegaT will detect that some directories are missing, and will present the directory selection dialog again so that new locations can be chosen.
