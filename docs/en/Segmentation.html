Segments

Translation memory tools work with textual units called segments. There are multiple ways to segment a text.

OmegaT first parses the text for structure-level segmentation. During this process it is only the structure of the source file that is used to produce segments.

Starting from version 1.6, OmegaT may segment by sentence. More about it in Sentence Segmentation. Sentence segmentation takes place after structure-level segmentation.

Segments created during either process cannot be changed (split or merged) during translation.

If you are not satified with the segmentation you will need to either modify the file externally and/or to modify the sentence segmentation rules. Once this is done you will need to reload the project for your modifications to be taken into account.

File Structure Segmentation

Text Files

Segmentation for text files in OmegaT occurs at line breaks. Line breaks are sometimes found at the end of sentences and sometimes at the end of paragraphs.

Paragraph-level segments, sentence-level segments, or segments that include parts of sentences will be created depending upon how your text file is formatted. Therefore, it is possible to force segmentation at any level by editing the line breaks in a file.

Don't forget to reload your project once you have modified your files.

Java Resource Bundles

Since 1.4.4 OmegaT supports Java Resource Bundle files. These files use one line per language item (character string) to be translated. The segmentation is very similar to that of text files, except that it is not possible to modify the segment boundaries.

Tagged (Formatted) Files

Relevant tagged files in OmegaT include: HTML, XHTML, OpenDocument, OpenOffice.org, and StarOffice. In general, these files contain two types of tags:

  • in line—usually defines the style of the text;
  • block level—usually describes its logical structure.

Segmentation in formatted files occurs at block level tags.

(X)HTML Files

Since version 1.6, OmegaT parses the internals of the (X)HTML file and extracts parts of the code that do not appear on the screen but that still need to be translated.

Typically this includes string data of human readable attributes like: abbr, alt to offer an alternative textual description of a picture, content, href, hreflang, lang to describe the file language, src, summary, title and value. Such strings appear as one segment.

Spreadsheets, tables, visual presentation

Text elements in a spreadsheet, a table or a visual presentation, among other documents a translator usually works with, will be segmented according to the object (cell etc.) that contains them.
