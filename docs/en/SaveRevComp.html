Translation: Save, Revision, and Compilation

Save

To save your work at any time, select Save Translation Memory from the File menu, or press <Ctrl+S>.

This will save your translation as an intermediate translation memory, in the file /omegat/project_save.tmx. To generate an up-to-date translation memory for the project, or target translated documents, compile the project (see the Compilation section).

Revision

To revise a translation, select its source file from the Project Files window and step through each segment by pressing <Enter> repeatedly, stopping in a segment to revise as necessary. When all segments of a file have been revised OmegaT will automatically switch to the next file in the list. Translators familiar with other translation memory applications will appreciate this convenient method of revision.

Compilation

Do not be intimidated by the computerese!

During translation with OmegaT the original source text is not overwritten. Each time a new translation is accepted (e.g. by pressing <Enter>) it is stored in computer memory along with all other unsaved translations until the project translation memory (project_save.tmx) is automatically saved (at approximately 20 minute intervals), or the user either performs a compilation to create the target documents or saves the translation memory. If the application is closed before any of these actions takes place all unsaved translations will be lost.

The process of compilation, done via Create Target Documents from the File menu, causes OmegaT to generate the translated documents and project TMX. In order to do this, any new unsaved translations are merged together into project_save.tmx with previously saved ones and then segments from the original text are replaced by these translations in the target documents.

A complete translation of a text does not have to be done before compilation is performed. It is possible to compile at any time. Of course, the resulting target documents will only be partially translated in this case.

Using Save Translation Memory from the File menu operates in the same way as for compilation, except that the target documents are not created.

Precautions

To avoid possible loss of data or disaster, here are a few suggestions:

  • Make regular backup copies of the file /omegat/project_save.tmx. A good idea would be to add a serial number or the date to each copy, for instance, /omegat/project_save.tmx-01 or /omegat/project_save.tmx-07-06-05. Backups can be transferred to other media such as floppy disk, CD-RW, or other hard disk if they are available. Then project_save.tmx can be conveniently restored from backup if something goes wrong.

    Please note that when translating formatted (OpenDocument, OpenOffice.org, StarOffice, XHTML, or HTML) files it is likely that project_save.tmx is much smaller than the original or final texts.

  • Make compilations at regular intervals and check that the compiled file contains the latest version of a translation. In particular, this should be done when first starting out with OmegaT.

  • Take particular care when making any changes to the files in /source in the middle of a project. Sometimes it might be necessary to make changes to source files, for example to remove hard line breaks from the middle of sentences. However, if the source file is modified after translation has begun, OmegaT may be unable to find a segment that has already been translated. Always backup the source file itself, or the whole project, before making any changes to the source file(s).

  • When creating a new project, do not forget that OmegaT will open its file selection dialog at the location it was last used. In most cases, this will be inside a project directory to select the omegat.project file. Make sure that the directory selected is not within an existing project.
