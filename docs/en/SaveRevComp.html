Save, Revision, and Compilation

  1. Save
  2. Revision
  3. Compilation

Save

OmegaT automatically saves translated segments at very shorts intervals of time. Nevertheless, never forget to save your work at regular times. For this, select Save Translation Memory from the File menu, or press <Ctrl+S>.

This will save your translation as an intermediate translation memory, in the file /omegat/project_save.tmx.

To generate an up-to-date translation memory for the project, or target translated documents, compile the project (see the Compilation section).

Revision

To revise a translation, select its source file from the Project Files window and step through each segment, stopping in a segment to revise as necessary. When all segments of a file have been revised OmegaT will automatically switch to the next file in the list.

Translators familiar with other translation memory applications will appreciate this convenient method of revision.

Compilation

Do not be intimidated by the computerese!

During translation with OmegaT the original source text is not overwritten. Each time a new translation is accepted (e.g. by pressing <Enter>) it is stored in computer memory until the project translation memory (project_save.tmx) is automatically saved, or until the user either performs a compilation to create the target documents or saves the translation memory.

If the application is closed before any of these actions takes place all unsaved translations will be lost.

The process of compilation, done via Create Target Documents from the File menu, causes OmegaT to generate the translated documents and project TMX from the data in the temporary translation memory (project_save.tmx).

A complete translation of a text does not have to be done before compilation is performed. It is possible to compile at any time. Of course, the resulting target documents will only be partially translated in this case.

Using Save Translation Memory from the File menu operates in the same way as for compilation, except that the target documents are not created.

Precautions

To avoid possible loss of data or disaster, here are a few suggestions:

  • Make regular backup copies of the file /omegat/project_save.tmx. A good idea would be to add a serial number or the date to each copy, for instance, /omegat/project_save.tmx-01. or /omegat/project_save.tmx-07-06-05. Backups can be transferred to other media such as floppy disk, CD-RW, or other hard disk if they are available. Then project_save.tmx can be conveniently restored from backup if something goes wrong.
  • Make compilations at regular intervals and check that the compiled file contains the latest version of a translation. In particular, this should be done when first starting out with OmegaT.
  • Take particular care when making any changes to the files in /source. in the middle of a project. Sometimes it might be necessary to make changes to source files, for example to remove hard line breaks from the middle of sentences. However, if the source file is modified after translation has begun, OmegaT may be unable to find a segment that has already been translated. Always backup the source file itself, or the whole project, before making any changes to the source file(s).

Please keep in mind that OmegaT creates a backup of a project's translation memory in the project_save.tmx.bak file in the project's /omegat subdirectory whenever a project is opened and before any modifications are made.

OmegaT is a robust application. It is not prone to crashes or data loss and corruption. However, you should take precautions against data loss when using OmegaT, just as with any other application.
