Editing tagged (formatted) texts

Working with tags

OmegaT displays the in-line formatting information of supported formats (at present HTML, XHTML and OpenOffice.org/StarOffice) as tags. The tags you see in OmegaT are not the same as those used by the original applications; OmegaT converts them internally, and converts them back at compilation.

Care must be taken with tags. If they are changed accidentally, the formatting of the final file may be corrupted. In worst cases, the file may refuse to open altogether: this is particularly the case with OpenOffice.org files.

Although you cannot add formatting of your own in OmegaT, you do have some degree of control over the existing formatting.

Tag manipulation in OpenOffice.org files

OpenOffice.org tags must be retained unchanged. You cannot delete these tags, duplicate them, or change the sequence. You can move or cancel the tags

Moving the tags

You can move the tags to a different part of the segment; they must, however, remain in the same order;

Cancelling the tags

By placing the tags next to each other you effectively cancel them.

For instance, if you change:
This text is in <f0>italics</f0>
to
This text is in <f0></f0>italics
you effectively remove the italic formatting.

Tag manipulation in HTML files

The start of formatting is marked by an opening tag of the form <xx>, and the end of formatting by a corresponding closing tag </xx>. Any changes made to formatting tags must be made to both tags in the pair.

You can delete, duplicate, change the order or nest the tags.

Deleting the tags

For example, to remove the italics from
This text is in <f0>italics</f0>

simply delete the tags <f0> and </f0>.

Duplicating the tags

For example,
This text is in <f0>italics</f0>, and so is <f0>this</f0>

applies the formatting of the word “italics”; to the word “this”.

Changing the order of tags

For example,
This text is in <f0>italics</f0>, this text is <f1>underlined</f1>
can be changed to
This text is <f1>underlined</f1>, this text is in <f0>italics</f0>

Nesting tags

For example:
This text is in <f0>italics</f0>, this text is <f1>underlined</f1>
can be changed to
This text is in <f0>italics and <f1>this text is also underlined</f1></f0>

You must not overlap tags.

For example:
This text is in <f0>italics, this text is also <f1>underlined</f0>, this text is underlined but not in italics</f1>

is not permitted and will corrupt your file if you try it. If you insert an opening tag and then open a second opening tag, you must insert the closing tag of the second opening tag before the closing tag of the first.

Validating tags

The validate tags function (Tools > Validate tags) detects tags to which changes have been made (whether deliberately or by accident), and indicates the affected segments.
