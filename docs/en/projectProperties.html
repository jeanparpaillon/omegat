Project properties

This dialog is used to modify the project folders and languages. It may be opened by pressing Ctrl+E or by calling Project -> Properties... from the main menu. The dialog contains:

  • Area for setting project source and target languages with drop-down menus
  • Segmentation settings flag
  • Area for setting project folders

All the fields are editable. You can either enter the languages by hand or use the drop down menu. You can type names of folders that do not exist yet, and OmegaT will create them for you.

Modifying project properties during the session

It is possible to modify the project properties during a translation session, however you have to bear in mind that changing the languages may render the currently used translation memories useless since their language may not match new languages anymore. If you change the segmentation settings, some already translated segments may get split or merged. This will practically return them to the "untranslated" status, even though the old and now not-matching translation is still in the translation memory. OmegaT may upgrade old translation memories that did not use sentence segmentation, but not vice versa, and not the project's translation memory if you change your mind in the middle of the translation.

In case you intend to modify project folders, remember that this will not move the files from old folders to new ones, so you risk ending up in a situation neither you or OmegaT know what the folders really mean, and which of them are used.

The safest way is to consider all settings as final once the translation has started - unless of course you realize a major mistake has been made. See the Preventing data loss section for ways to protect your work.
