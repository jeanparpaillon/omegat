User Interface Items

The OmegaT user interface consists of a number of items. Below we first summarize all of them, and then describe each of the items in more detail.

Windows you'll use most of the time

  • OmegaT main window with 3 panes:
    • Editor pane where you type and edit the translation
    • Match Viewer pane that shows the most similar segments from translation memories
    • Glossary Viewer pane with the terminology translation
    • And also the Main Menu where you can access most of OmegaT functions
    • As well as the Status Bar at the bottom of the window to look for workflow related messages
  • Project Files window with the list of your translatable files
  • Search window used to find specific segments in the project

Windows you'll use sometimes

  • Tag Validation window to detect mistakes in the formatting of translated documents
  • Help browser to read the user manual

Dialogs you'll use to change your project settings

  • Project Properties to modify the project folders and languages
  • Font to modify the font OmegaT uses for displaying source, translation, matches and glossary terms
  • File Filter to adjust the handling of supported file formats
  • Segmentation to change the way your text is segmented into sentences
  • Editing Behaviour to change how OmegaT behaves when you iterate between the segments

OmegaT Main Window

Main window consists of three panes, main menu and a status bar.

You can change the position of any pane or even undock it to a separate window by dragging the pane by its name. Depending on the pane status, 5 different signs can appear at the top right corner of the pane. Putting the mouse pointer over each sign should display a tooltip.

reduces the pane to show only its name at the bottom near the status bar.
makes the pane take all the available window space.
puts the pane back where it was before before it was minimized or maximized.
detaches the pane from the main window to a separate window of its own.
puts the pane back within the main window.

It is also possible to put the panes so that they overlap. In this case the panes will display a tab at their top. Clicking on the tab will put the pane on the foreground. The separators between the panes can be dragged to resize panes.

Editor Pane

Here you'll type and edit the translation. Editor pane displays the text of the partially translated document: the text already translated is displayed translated, and the text not yet translated is displayed in the original language.

The displayed text is split into segments, and you may scroll through the document and double-click on any segment to edit it.

One of the segments is active. It is the only one displayed in two parts: the upper part is in the original language, in bold characters with a green background color, the lower part is the editing field, delimited by two markers: the leftmost marker is "<segment nnnn>" where "nnnn" is a number of the segment in the project, the rightmost marker is "<end segment>". You should use the upper part as a reference and overwrite or modify the contents of the editing field to contain the translation. 

This sentence has been translated.

This sentence has been translated too.

Cette phrase est en train d'Ãªtre traduite.
<segment 0003>[Here you should type: This sentence is being translated]<end segment>

Depending on the editing behaviour the editing field for the untranslated segment may be empty or contain different text when you move to it:

  • The source text
  • The translation of the string most similar to the one you are going to translate

When you move to another segment, the translation is validated and stored.

If you want the segment to stay untranslated, simply make the editing field empty by removing all the text (select all with Ctrl+A and delete with Del).

Depending on the options you set in Editing Behaviour settings, OmegaT may remember a translation when it is identical to the source. It is useful for documents that contain trade marks, names or other proper nouns, or parts in a third language that does not require translation.

See Translation Editing for more details.

Fuzzy Matches

Match viewer shows the most similar segments from translation memories, both from project translation memory created while you translate your project, and from legacy translation memories you import from your earlier jobs, or you may receive from your client or translation agency.

When you move to the next segment, the first fuzzy match (the one with the best matching percentage) is automatically selected. You may select another one by pressing Ctrl+2, 3, 4, or 5, depending on the number of available matches.

To use the selected match in your translation, use Ctrl+R to replace the target field with that match or use Ctrl+I to insert it at the cursor position.

The matching percentage is roughly calculated by taking the number of common words in the matched and the matching segment and dividing by the number of words in the longest of the two.

Selected fuzzy match

  • Will be highlighted in bold
  • Words that are missing from the segment you translate will be blue
  • Those words near blue ones or in positions not similar to original will be green

Consider the segment: 

   Information about Java & OmegaT

The matching segment:

   General notes about Java & OmegaT

will be colored: 

   General notes about Java & OmegaT

and the percentage should be around 3/5=60%

Glossary

Glossary pane features terminology translation loaded from glossary files. It shows translation of terms found in the current segment, but it only shows it for reference purposes, and does not allow to insert or replace the term with its translation. OmegaT also supports multi-word terms, but in a very basic manner: if both words of a term are found in current segment, the term is shown.

The format for terminology in the Glossary pane is 

   'source term' = 'term translation'
   General comment about the term, if any

Main Menu

Obviously, you can use main menu to access all OmegaT functions. The most often used functions are accessible with keyboard shortcuts, so once you get accustomed to them, there will no longer be any need to browse through the menus while translating.

See the Main Menu appendix for a full description of all menu's and menu items.

Status Bar

The status bar displays workflow related messages at the bottom of the main window. This bar gives the user feedback messages about specific operations that are in progress. It also displays the number of fuzzy and glossary matches for the current segment.

Other Windows

Project Files

Project Files window lists the project files and displays other project information. It is displayed automatically when OmegaT loads a project. When the project is open, you may use Ctrl+L shortcut in main window to close it when it's visible, or open it when it's hidden.

Project Files Window features:

  • Clickable listing of all the translatable files in the project. These are the files present in the source files folder, that OmegaT can handle. Clicking on any file will bring it to the Editor Window for translation.
  • Number of segments in each file is shown next to file name.
  • Number of unique (different) segments in the whole project.
  • Number of unique segments you have already translated. It is updated each time you translate a segment.

The set of Unique segments is computed by taking all the segments and removing all the duplicate segments equal up to letter case ("Run" and "run" are thought different).

By looking at the difference between "Number of segments" and "Number of unique segments" you get a rough idea of how much repetitions you have in the text. The numbers do not give a clear indication of how important are the repetitions: segments could be relatively long sentences repeated a number of times (very lucky!) or simply keywords filling a table (not as lucky...)

It is possible to modify the number of segments/unique segments by modifying the segmentation rules. Such modifications should _never_ take place after you started translating the project. See the segmentation rules for more information.

You may add source files to the project by clicking on the "Import Source Files..." button. It copies the selected files to the /source directory and reloads the project to load the new files.

Search Window

You'll use search window to find specific segments in the project. There can be several search windows open simultaneously. To open a new one, use Ctrl+F in main window.

The window consists of:

  • Search field to enter search string or keywords
  • A number of flags and radio buttons to change search settings
  • Clickable result display space

Pressing the search button after entering a string in the search field will display all the segments in the project that include the searched string. As OmegaT handles all the equal segments as one entity -- if you change one of them, all the same segments get updated, only the first unique segment is shown.

The segments are displayed in order of appearance in the project. Translated segments will be displayed with the original text at the top and the translated text at the bottom, non translated segments will be displayed in original text only.

If the string is found in one of the project files, you may click it to bring it up in the Editor to modify. Then you may switch back to Search window and go to the next found segment to modify it too. Such a pattern is useful to do massive terminology updates.

There are two types of search available:

  • Exact searches look for the specified query string within the current project or in any directory or directory tree. Exact search supports the wildcard characters * and ?. The '*' character matches zero or more characters (the search term 'run*' would match 'run', 'runs', and 'running', for example). The '?' character matches exactly one character ('run?' would match 'runs' and 'rung', for example, but not 'run' or 'running'). This operation searches both the source and translation.
  • Keyword searches examine the current project and return all the segments containing all of the query words, in any order.

You may also search within any directory, but bear in mind that OmegaT can search only in files it supports.

Tag Validation

Tag validation window

The tag validation window is used to detect mistakes in formatting of the translated documents. To open the window, use Ctrl+T.

The window features a 3 column table with:

  • Clickable link to the segment with possible tag error
  • Contents of the original segment
  • Contents of the translated segment

Clicking on the link makes the segment active in the Editor.

Tag errors

A possible tag error is one of the following:

  • A tag was in the source segment but has been erased from the translated segment.
  • A tag was not present in the source segment but has been inserted by mistake in the translated segment.
  • The tags order has been modified in the translated segment.

Hints for tags management

Tags generally represent some kind of formatting in the original text. Simplifying the original text formating greatly contributes to reducing the number of tags. Unifying used fonts, font sizes, colors etc should be considered, if possible, to simplify the translation and reduce the number of possible tag errors.

Take a look at the Tag operations section to see what can be done with tags.

Remember, if tags bother you and formatting is not extremely relevant to the job at hand, removing most of the formatting from the source document will greatly ease the translation.

If you need to see the tags in OmegaT but do not need to retain most of the format in the translated document you are free not to add the tags to the translation. In this case pay extra attention to tag pairs. If only one tag out of the pair remains, it will corrupt your document's formatting and makes a translated OpenOffice.org document invalid or a translated HTML file unreadable.

Since tags are included in the text itself, it is possible to use segmentation rules to create segments that include less tags. This is an advanced feature and should not be used by beginning users.

Help Browser

The help browser (which displays this manual) can be called by pressing F1 or calling Help -> User Manual... in main menu.

In the window you'll see the manual and two buttons: Back and Contents.

The user manual is an HTML document with links to different chapters. Clicking on a link as you would do in a web browser brings the desired page to the front.

The user manual is located in doc subfolder under OmegaT installation folder, so you may view, for instance, English documentation, by opening the doc/en/index.hmtl file in your browser. This way you will be able to follow external links too, as built-in help browser does not accept external internet links.

Dialogs

You'll use the dialogs below to change OmegaT and project settings. All of them are described in separate sections, below we just summarize what they can be used for and how you may call them up.

  • Project Properties is used to modify the project folders and languages. You may open it be pressing Ctrl+E or by calling Project -> Properties... from the main menu.
  • Font Settings is a dialog to modify the font OmegaT uses to display source, translation, matches and glossary terms. It is called through Options -> Font... main menu item.
  • You may use File Filter Options to modify the files that should be handled by each of the available filters.
  • The Segmentation Settings dialog allows you to change the way your text is segmented into sentences. Note that sentence segmentation is enabled/disabled in the Project Properties dialog.
  • The Editing Behaviour dialog changes how OmegaT will behave when you iterate between the segments.
