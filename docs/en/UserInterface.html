The graphical user interface of OmegaT consists mainly of an Editor window and a Match and Glossary Viewer window.

This user interface is localized to a number of languages and if the language of your computer environment is available it should be displayed in this language. See Available languages for OmegaT 1.6 for the languages available in this version and Changing the user interface language settings to have OmegaT display an available language of your choice.

Editor Window

The Editor displays the text currently in the process of translation. The state of each segment is clearly indicated in the editor pane. Untranslated segments are displayed in the source language and translated segments are displayed in the target language.

The only exception to this is the active segment where translation and the revision take place. It has two features: the segment is displayed in an editing field and bracketed by <segment number> and <end segment> tags; and the source language text is displayed above the editing field in highlighted bold characters for easy reference;

Warning! It may take a noticeable amount of time for OmegaT to update the display after you have resized the window, especially when a big project is opened. Generally speaking, it is better to resize the window before opening a project.

Warning! Closing the editor manually (with the mouse for example) will automatically close your project.

Match and Glossary Viewer

The Match and Glossary Viewer is automatically displayed when OmegaT is started. It can be opened and closed with the File > Match/Glossary Viewer... menu item (<Ctrl+M>).

It is divided into two panes. The Match pane displays fuzzy matches (previously translated segments with a certain degree of similarity to the active segment). The Glossary pane displays information from the glossary file(s), if one exists.

You can modify the respective sizes of the panes by dragging the separation up or down. You can also use the small arrows on the right of the separation to make a pane use the whole window.

Project File Window

This window automatically opens when a project is opened. Access to it is available through the View > Project Files... menu item when it is closed (<Ctrl+L>). You can dismiss the window by pressing <Esc>.

The Project Files window contains a list of files for translation in the current project. Select a file for translation by clicking on its name in the window.

The window also displays the number of unique segments in each file, the total number number of unique segments in the project and the number of translated segments. The number of translated segments is updated each a segment is translated.

A Import Source Files button is enabled to assist you in importing source files at project creation. This function is similar to the Import Source Files... menu item below. You can use it while working on a project.

Search Window

This window opens when called through the Edit > Search Project... menu item (<Ctrl+F>). See Searches for more information.

To keep track of previous searches calling a search window will always create a new window.

Tag Validation Window

This window opens when called through the Tools > Validate Tags... menu item (<Ctrl+T>). If tag errors are detected, the window will display a 3 column table with:

  1. the number of the segment where the error was detected
  2. the source content of the segment
  3. the target content of the segment
Clicking on the segment number will bring it to the front in the Editor window for verification.

See Tag Validation for more information.

Help Browser

This window opens when called through the Help > User Manual... menu item (<F1>). It displays the HTML version of this manual in a local browser that will not access external links to the internet.

You can display the User Manual in a normal web browser and have access to external links by opening the index.html file located in the /docs/ directory of the OmegaT application directory.

Project Properties Window

This windows opens when called through the Project > Properties... menu item (<Ctrl+E>) or at project creation. See Create and open a new project for a visual image.

All the fields are editable. You can either enter the languages by hand or use the drop down menu. You can type the names of folders that do not exist, OmegaT will create them on the spot.

Warning! You can enable or disable the sentence segmenting function from this window. Modifying this setting while working on a project can be extremely damaging. See Precautions against data loss! for ways to protect your work.

Files Filtering Settings

This window can be accessed only when no project is opened. It opens when called through the Options > File Filters... menu item. See File Filters for more information.

Segmentation Settings

This window can be accessed only when no project is opened. It opens when called through the Options > Segmentation... menu item. See Sentence Segmentation for more information.

Sentence level segmentation can be enabled through the Project Properties window. See above The Project Properties Window for more information.

Editing Field Options

This window can be accessed at all times. It is called through the Options > Editing field... menu item. See Editing Field Settings for more information.

Font Settings

This window can be accessed at all times. It is called through the Options > Font... menu item. See Language Character Sets and Fonts for more information.

Warning! It may take a noticeable amount of time for OmegaT to update the display after you have changed the font setting, especially when a big project is opened. Also, some fonts behave better for some language pairs than for others. Generally speaking, it is better to change of font before opening a project.

Menu items

All of OmegaT's functionality is available through the menu bar at the top of the Editor window. Most functions are also available via keyboard shortcuts.

The shortcuts given here are for those operating systems with a Control (Ctrl) key (UNIX, Windows, etc.). For Mac OS X use the Apple key (cmd) in place of Ctrl.

Project Menu:

New
Creates a new project (disabled when a project is open). See Create a project.
Open <Ctr+O>
Opens a previously created project (disabled when a project is open). See Open a project.
Import Source Files...
Copies the selected files to the /source/ directory (disabled when no project is open). See Create a project and Project Files Window above.
Reload
Reloads the project to take external modifications into account (disabled when no project is open).
Close
Close project after saving the translation (disabled when no project is open).
Save <Ctr+S>
Saves the translation into the translation memory (disabled when no project is open). OmegaT automatically saves the translation at very short intervals of time.
Create target Documents
Creates the target documents based on the translation memory (disabled when no project is open). The documents will be located in the /target/ directory.
Properties <Ctr+E>
Displays the project details for modification (disabled when no project is open). See Project properties window above.
Quit <Ctr+Q>
Quits OmegaT after saving the translation.

Edit Menu:

Items that one can find in most applications (copy/cut/paste) are not displayed in this menu but are available with your system shorcuts.

Undo Last Action <Ctr+Z>
Goes back to the status before the last editing action was taken. Does not work once the modified segment has been validated.
Redo Last Action <Ctr+Y>
Goes back to the status before the last editing action was cancelled. Does not work once the modified segment has been validated.
Overwrite Translation <Ctr+R>
Overwrites the whole target segment with the currently selected fuzzy (fuzzy#1 default value).
Insert Translation <Ctr+I>
Inserts currently selected fuzzy at the cursor position (fuzzy#1 default value). This function will never overwrite a portion of the target segment.
Overwrite Source <Shift+Ctr+R>
Overwrites the whole target segment with the source.
Insert Source <Shift+Ctr+I>
Inserts the source at the cursor position.
Search Project... <Ctr+F>
Creates a new search window. See Searches.
Select Fuzzy Match #1 <Ctr+1>
Selects the first fuzzy displayed in the Match and Glossary Viewer. See Matches and Match and glossary viewer above.
Select Fuzzy Match #2 <Ctr+2>
Selects the second fuzzy displayed in the Match and Glossary Viewer.
Select Fuzzy Match #3 <Ctr+3>
Selects the third fuzzy displayed in the Match and Glossary Viewer.
Select Fuzzy Match #4 <Ctr+4>
Selects the fourth fuzzy displayed in the Match and Glossary Viewer.
Select Fuzzy Match #5 <Ctr+5>
Selects the fifth fuzzy displayed in the Match and Glossary Viewer.

Go to Menu:

Next Untranslated Segment <Ctr+U>
Shifts the Editor focus to the next segment that has no equivalent in the translation memory.
Next Segment <Ctr+N>, <Return>, <Enter> or <Tab>
Shifts the Editor focus to the next segment. When the current segment is the last of a file, the next segment will be the first segment of the file displayed next in the Project Files Window.
Previous Segment <Ctr+P>
Shifts the Editor focus to the previous segment. When the current segment is the first of a file, the previous segment will be the last segment of the file displayed before the current file in the Project Files Window.

View Menu:

Match/Glossary Viewer... <Ctr+M>
Displays or dismises the Match/Glossary Viewer. See Match and glossary viewer above.
Project Files... <Ctr+L>
Puts the Project Files Window to the foreground (can be dismissed with <Esc>). See Project Files Window above.

Tools Menu:

Validate Tags <Ctr+T>
Checks for missing or displaced tags in formatted files. Will display a list of segments with tag mistakes. See Tag Validation and Tag Validation Window above.

Options Menu:

Use Tab To Advance
Sets segment validation key to <TAB> instead of the default <Return> or <Enter>.
Font...
Presents the user with a dialog to modify the text display font. It is better to change of font before opening a large project. See Language Character Sets and Fonts and Font Settings above.
File Filters...
Displays a dialogue to configure file handling and parsing (disabled when a project is open). See File Filters Setup and Files Filtering Settings above.
Segmentation...
Displays a dialogue to configure text segmenting (disabled when a project is open). You can modify text segmenting while the project is open by selecting Properties in the Project menu. See Rules Setup and Segmentation Settings above.
Editing field...
Displays a dialogue to configure target field contents. See Editing Field Settings and Editing Field Options above..

Help Menu:

User Manual... <F1>
Opens a separate window with this manual for easy reference. See Help Browser above.
About...
Displays credit and user license information.

Status Bar

A status bar is visible at the bottom of the Editor. This bar gives the user feedback messages about specific operations that are in progress. It also displays the number of fuzzy and glossary items matches.
