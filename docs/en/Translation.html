Create a new project

Select Project -> New... from the menu.

Navigate to the folder where you'd like to save the project files, and type in a name for the translation project. This name will also be the name of the project folder, and all project files will be stored in that folder.

OmegaT will prompt you to change or accept the project folders to be created.

You may simply accept the location of the subfolders (directories), but you should make sure that source and target language codes are correct. Select the language or language-and-country codes from the dropdown-list or type them in by hand. Make a mental note of the location of the "target" subfolder, for this is where target documents will be exported to.

If you want OmegaT to segment by sentence instead of by "paragraph", it's a good idea to double-check the Segmentation Rules.

Project loading and files

When you click OK to accept the project setup, OmegaT will prompt you to import source documents. You can import individual files, but you can also import entire directory trees (with all files in all subfolders). If you've accidentally imported the wrong documents or too many documents, you can simply remove them from the "source" folder of your OmegaT project.

Remember, OmegaT can only translate OpenDocument files, OpenOffice.org documents, HTML, plaintext and certain Java files. Other files will be ignored by OmegaT.

Make sure View -> Project Files... is enabled, and then select Project -> Reload to view a list of all files to be translated. OmegaT loads the first file on the project list by default.

If OmegaT indicates that a newly created project is empty after the source files for translation have been copied into the /source directory, then the files are probably in an unsupported format (MS Word. etc.). All files must be converted to a supported format, such as OpenDocument text or OpenOffice.org writer, for OmegaT to work with them. See Source Document Files for details.

If the original source documents intended for translation are in an unsupported file format remember to convert them to OpenDocument or other supported format and place them in the /source sub-directory of the project. It is usually only necessary to place the converted source documents in this directory.

The originals can be kept in a different directory. Any documents that are included in this directory that are in an unsupported format will be copied to the /target directory without modification when the project is compiled.

If you cannot open a translation memory received from an external source (colleague, customer, etc.) it may be in an unsupported TM format. If it is a TMX file (file extension .tmx), it is probably a higher version than currently supported. OmegaT only supports TMX Version 1.1 (Level 1) presently. The most recent TMX version is 1.4b. See Translation Memory and Glossary Files for more information.

or Open an already existing project

Select Project -> Open... from the menu.

Navigate to the folder where you'll see the OmegaT icon associated to a folder named after your project. Click on that folder. The project will load automatically.

Basics of OmegaT

To translate text just overwrite it in the active segment's editing field.

Press <Enter>, <Return> or <Ctrl+N> to accept a translation and proceed to the next segment. It is possible to use <Tab> as an alternative. To activate this functionality, enable the Use TAB to Advance menu item in the Options menu.

When a project is opened the active segment defaults to the first segment of the first file in the Project Files list. To jump quickly to the next untranslated segment of the project, press <Ctrl+U> or choose the Next Untranslated Segment menu item from the Edit menu.

Move to the:

  • next segment by pressing <Enter>, <Return>, <Ctrl+N>, or <Tab>
    (Tab Advance must be enabled in the Options menu for use);
  • previous segment by pressing <Ctrl+P>;
  • next untranslated segment by pressing <Ctrl+U>.

In some case the source and target segments will be the same even after translation. In such cases OmegaT will consider these segments as untranslated since there is no way to know if the segments' translations have been completed. Segments are considered to be untranslated when source and target segments are the same. Such segments will nevertheless be present in the final translation.

Open an appropriate application with the original source text in it to see the layout of text during translation.

Navigation

Movement to a different segment within the same file is accomplished by dragging the vertical scroll bar of the window up or down, then mouse click inside the desired segment to activate it.

To access a different file within the same project, select it in the Project Files window by clicking on the filename of the desired file to open in the editor. By default, the first segment at the beginning of the file will be the active one. Open the project files list window with File > Project Files..., if it is not already visible.

To return to a segment later, insert a marker in the segment. Then it will be possible to find it using the search functionality. It is a good idea to include the segment number with the marker. The reason being that if the segment is a 100% exact match to other segments the marker will be reproduced in all similar segments due to automatic insertion. The only way to locate the right segment is to compare the marker, which now includes a number, to the active segment number (see Searches: Keyword and Exact).

See also: General Hints and Tips.

Editing

OmegaT's editing functions are only applicable to the target text of the active segment. This is the only text that may be edited at any one time. However, text in any segment outside the active one can be marked, copied, or pasted.

The usual mouse operations, cursor keys, cursor key shortcuts, and shortcuts (select, cut, and paste) are supported. These are likely to vary between operating systems though.

Undo and Redo operations are possible with <Ctrl-Z> and <Ctrl-Y>, respectively.

When working on a formatted file, the editor field will contain OmegaT specific tags that refer to the file's internal formatting structure.

It is not possible to add or change text formatting during translation. The formatting of the underlying structured file format, such as OpenDocument, OpenOffice.org, XHTML, or HTML (the only ones supported at present), is retained. Only a limited degree of control can be exercised over it. Refer to Tagged (Formatted) Files Text Editing for details.

Markers, edition and navigation

One way to add formatting is to insert a marker in the text, where formatting is desired (using "##", for instance) and then format the text in another application (e.g. OpenOffice.org) using the target documents that result from project compilation.

After moving to a segment in a different part of a large project, the location of the previous segment that was being worked on may be difficult to locate in order to return to it. To enable quick segment location, place a unique marker in the current working segment, such as '###xxxx' (where xxxx. is a unique value, like the current segment number) then press <Enter>, to confirm the change to the segment before moving to a different location. A search for the unique marker will cause it to be the active segment again.

To insert a marker in your translation, to indicate an unresolved query or for some other reason, do not use ?. or *. These characters are used in searches (as wildcards) and are therefore impossible to find with <Ctrl-F>.

Matches

When a segment becomes active, OmegaT scans the current project translation memory file and any other translation memories found in the /tm directory of the project for matches.

If an exact match is found in a project's TMX it is automatically inserted into the target text field of the active segment and any other matching segments in the same document. It must be understood that close matches such as "Hi!" and "Hi" are 100% matches (the words are the same), but not exact matches and thus will not be inserted automatically. The best five fuzzy matches that are located will be displayed in the Matches pane of the Match and Glossary Viewer window.

If there are fuzzy matches for the current segment it is possible to:

  • select the most useful fuzzy with <Ctrl-(nb)> where (nb) is the number of the fuzzy as displayed in the Match and Glossary Viewer
  • insert the selected fuzzy match in the editor field with <Ctrl-I>
  • rewrite the whole field with the selected fuzzy match with <Ctrl-R>

Editing Field Settings

Starting with OmegaT 1.6 you can configure the contents of the target editing field.

The Option menu item Editing field allows the following configurations:

Contents of the target field
You are presented with a 3 radio buttons set with possible values:
  • Leave the segment empty
  • Source text
  • Best fuzzy match
Source text was the default behavior until OmegaT 1.6. You can now choose either an empty editing field or a field with a fuzzy match inserted.
If you select Best fuzzy match, only the best fuzzy match with a similarity score above the set level will be inserted. Fuzzy matches below this level, even if present, will not be inserted and the field will be left empty.
Minimal similarity
This setting allows you to set the level from which OmegaT considers a fuzzy valid for insertion in the target field. The highest the number is, the most similar the matched segment will be the current source segment.
Prefix
To make sure you notice the automatic insertion of a "best match" in your target field, OmegaT inserts a string at the beginning of the best match. You can set the value of the string here but it cannot be set to be empty. Since the best match is always selected by default (it is the first one), you can use Ctrl+R to rewrite the automatically inserted string with the match itself, which will remove the "best fuzzy" prefix part.
Allow target to be equal to source
This setting allows you to create translation units with the target part equal to the source part. This can be useful when a segment needs not be translated but you still want to insert it into the translation memory. Default behavior is to consider segments where target is identical to source as untranslated. See Basics of OmegaT above.

Translated files

Create translated files at anytime during the translation process by compiling the project documents with the Create Target Documents menu item from the Project menu. It is not necessary to have all files and segments translated before using this function. The translated segments will be written to the files and any remaining untranslated segments will be written out untouched.

This can be useful, for example, to perform a partial review of the translation. The target files are created in the /target subdirectory of the project (for details on files and directories see OmegaT Files and Directories).

Any changes made externally to the files in /target will be overwritten and lost at the next creation. If the translation of the target files is not satisfactory, redo the translation from within OmegaT (see also Translation: Save, Revision, and Compilation, General Hints and Tips and Keyboard Shortcuts).

Problems!

Errors: in general, if errors or program bugs are encountered or suspected there is a log file (log.txt. in the root directory of a project that can be checked. Errors and bugs can be reported to the users group or development team. See Important Notes for more information.
