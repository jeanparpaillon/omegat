About this manual

This manual is a work in progress. For further information about OmegaT, please visit the OmegaT home page. Should you have any questions or comments on the manual, positive or negative, please send them to the author. Feedback is welcome!

OmegaT was developed on the Linux platform, and the development/documentation team does not use Windows. Although this documentation is intended for OmegaT users on all platforms, some terminology may therefore be a little unfamiliar. So, when you see the Linux forward slash /, think “\”, and do a mental translation to “folder”; whenever you read “directory”.

What is translation memory?

Translation memory is used during the translation of documents from one language to another. It assists you by storing pieces of text in a database (the “translation memory”) so that when you next encounter the same piece of text (a “segment”), or part of it, you can see immediately how you translated it in the past. For translation memory to be really useful, you must use it for long enough to build up a database of segments and their corresponding translations.

What's special about OmegaT?

There are many translation memory programs on the market. OmegaT's particular features are:

  • it runs on the Java Runtime Environment, and can therefore be used on a number of platforms, including Windows, Linux, and Mac OS X;
  • it is free;
  • it is open-source, meaning that users are free to distribute and modify it;
  • it is designed with speed in mind, rather than support for complex data formatting and multiple file formats;
  • it supports the OpenOffice.org file format directly.
