Source Document Files

OmegaT supports the following source document file formats for translation: plaintext; HTML; XHTML; OpenDocument; OpenOffice.org; StarOffice; and Java resource bundles.

These file formats are automatically detected based upon their filename extensions. Make sure that only files with the following extensions are used:

.txt, .txt1, .txt2, .utf8;plain text
.htm, .htmlHTML, XHTML
.od*OpenDocument (exact extension depends on file type)
sx*OpenOffice.org or StarOffice (exact extension depends on file type)
.propertiesJava resource bundles

OpenOffice.org possesses excellent import and export filters for file format conversion–this includes filters for MS Word, Excel, and Powerpoint. Therefore, files in different formats can readily be converted to the OpenDocument (OpenOffice 2.0) or OpenOffice.org native formats, then translated in OmegaT, and finally converted back to the original format again.

It may also be possible to use other wordprocessors and editors to create OpenDocument or OpenOffice.org file formats. AbiWord and KWord (KDE Environment) are two possibilities. Unfortunately, support for MS formats is not necessarily at the same level as with OpenOffice.org.

Plain Text

For plain text files the following encodings are supported:

  • Latin-1, ISO-8859-1 (.txt1);
  • Latin-2, ISO-8859-2 (.txt2);
  • UTF-8 (.utf8);
  • the default operating system text encoding (.txt).

HTML/XHTML

HTML and XHTML encodings are automatically detected when files of these types are opened. Please note that, by default, these file types are saved as UTF-8 encoded files.

OpenDocument

OpenDocument is an OASIS document file format that supercedes OpenOffice.org 1.x format.

Supported versions include: OpenDocument 1.0

To open OpenDocument formatted source or target files for review or other operations external to OmegaT requires OpenOffice.org 2.0 or greater, StarOffice 8.0 or greater, or another OASIS OpenDocument compliant wordprocessor or editor.

OpenOffice.org/StarOffice

Supported versions include: OpenOffice.org 1.x and StarOffice 6.x.

Java Resource Bundles

These are Java localization files used by user interfaces (i.e OmegaT). They have a .properties file name extension by default.

The default character encoding expected for this format is Latin-1 (ISO 8859-1). To use characters that are unsupported by this format they must be entered as Unicode escape sequences (i.e. '\u' followed by a four digit code).

Filename Extension Mapping

Extension mappings to respective formats are described in File Filters. The method used in previous versions (via omegat.prefs) is now obsolete.
