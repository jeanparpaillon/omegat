Getting Started with a Sample Project

The procedure presented here is for explanatory purposes only; variations on this procedure will most likely be necessary in practice.

Create a New Project

  1. Launch OmegaT.
  2. Create a new project by selecting New Project... from the File menu in the editor menubar.
  3. In the file selector dialog that appears, navigate to a suitable directory where the new project directory and its files are to be created.
  4. In the File Name text entry area, enter a project name (e.g. ABC), then press Save to confirm it.
  5. Another dialog will appear indicating the directory paths for the project. In the dialog, OmegaT will propose the creation of a new directory with the project name ABC that was selected. Five other default subdirectories are also shown. The locations of these default directories can be modified, but for the sake of this example just accept the values given (these are explained in the Project Files and Directories manual page).
  6. Enter language codes for the source and target files or select them from the drop down lists. These codes are required by the TMX standard. As used in OmegaT, the language code is composed of a 2 character language code (ISO 639), in lowercase, and an optional 2 character country code (ISO 3166), in uppercase. The codes must be separated by a hyphen in accord with the requirements of the standard.

    It is also possible to use locale codes in place of the language codes already specified. Locales codes are also made up of a language code and a country code. The difference between the two types of codes is their application and syntax. Locale codes are more commonly associated with software localization and computer systems and other software issues. A locale code has its codes separated by an underscore instead of a hyphen.

    The actual codes are particularly important if you send, receive, or exchange translation memories with external entities (customers, colleagues, organizations, etc.); in which case you should be careful to follow required standards for a particular format and application. In any event, language codes will be used in the TMX; locale codes are automatically written to the TMX as language codes.

  7. A project directory (also referred to as the project root directory), with the ABC name selected previously, will now be created. Within the project root directory are five subdirectories:

    1. /glossary - glossary files;
    2. /source. - source document files;
    3. /omegat. - project files;
    4. /target. - target document files;
    5. /tm. - translation memory files.

    See Project Files and Directories also.

Translate a File

  1. Place a text file (with extension .txt. to translate inside the /source directory. The .txt text file must be in the default operating system encoding. To use a Unicode encoded file (UTF-8) rename the file with .utf8. as the file extension instead.
  2. Open the newly created project. Select Open Project... in the File menu of the editor (OmegaT will automatically look in the directory last opened). The project root directory is displayed with the OmegaT icon--this indicates that it is a translation memory project.
  3. Open the project by selecting the project directory marked with the OmegaT icon (double-click or select and confirm with Open). Do not try to open a file, it won't work! It might crash OmegaT also.
  4. The project is now open and the file's text is displayed in the editor window.
  5. Place the cursor in the target field of the first segment, between the tags <segment 0001> and <end segment>. Type in a translation and delete the original text. Press <Enter> to accept the change.
  6. Repeat the process for each segment until the translation is complete.

Create the Translated File

  1. Select Save Translation Memory followed by Create Target Documents from the File menu. At this point OmegaT may be closed, if desired. Just choose Quit from the same menu.
  2. The translated file will be located in the /target directory.
