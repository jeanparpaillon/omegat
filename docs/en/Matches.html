Matches: exact matches, fuzzy matches

When a segment becomes active, OmegaT scans the current project translation memory file and any other translation memories found in the /tm directory of the project for matches.

Types of matches

Exact matches
These matches are automatically copied into the segments they are identical to without being user selected. This only happens if the match belongs to the project's translation memory (i.e. file project_save.tmx located in the /omegat/ directory). Any exact match found in other translation memory files located in /tm/ will not be automatically inserted, they must be deliberately inserted.
Fuzzy matches (100%)
Matches such as these differ by a punctuation mark, a number, formatting, or other similar characters. These are not automatically inserted.
Fuzzy matches (Less than 100%, more than 30%)
These differ by wording, but still have many common words with the active segment. Again, these are not automatically inserted.

The percentage match value indicated by OmegaT is a general indication of how much of the matched string is present in the source segment. It is only of technical interest.

Using Matches in your work

The best five fuzzy matches that are located will be displayed in the Matches pane of the Match and Glossary Viewer window.

Press <Ctrl+1>, to select the first match, and <Ctrl+2> to select the second, and so forth. The selected match is then displayed in bold. The name of the translation memory file containing the match shown is also displayed when the matching segment belong to an external translation memory.

A selected match can be pasted into the active segment by using <Ctrl-I> to insert the match at the current cursor position or <Ctrl-R> to substitute it for the active segment text.

The differences between the active source segment and the fuzzy match are indicated by coloring the match: black are those words that are included in the source segment together with their neighbors, green words are included but without their neighbors, and blue words are missing from the active segment.

Matches Options

Starting with OmegaT 1.6 you can configure the contents of the target editing field.

The Option menu item Workflow allows the following configurations:

Do not copy source text into target field
The default behavior is to have the source text into the target field. You can leave the field blank for immediate writing by setting this option.
Insert best fuzzy match into target field
This option will automatically insert the best fuzzy match if the field is empty, or rewrite the field if the source text is present.
Minimal similarity
This setting allows you to set the level from which OmegaT considers a fuzzy valid for insertion in the target field. The highest the number is, the most similar the matched segment will be the current source segment.
Do not insert "[best fuzzy match]" before the best fuzzy match itself
To make sure you notice the automatic insertion of a "best match" in your target field, OmegaT inserts a [best fuzzy match] string at the beginning of the inserted string. Since the best match is always selected by default (it is the first one), you can use Ctrl+R to rewrite the automatically inserted string with the match itself, which will remove the "[best fuzzy match]" part.

Glossary items

OmegaT has an automatic glossary function. By placing an existing glossary, in the /glossary directory after creating a project, it will automatically be detected when a project is opened. Terms in the active segment which are found in the glossary file(s) are displayed in the Glossary pane of the Match and Glossary Viewer window.

The glossary function only finds exact matches for the glossary entry (not inflected forms, etc.).

New terms can be added manually to the glossary file(s) during translation (for example in a text editor), but newly added terms will not be recognized until the project is reloaded.
