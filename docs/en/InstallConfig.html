Java Runtime Environment

OmegaT requires a Java Runtime Environment. (also known as JRE). Essentially, if your system will run Java programs it will run OmegaT. Java Runtime Environments are available for the following platforms: Windows 95/98/2000/ME/XP, UNIX, Linux, and Solaris (both x86 and Sun SPARC). Mac OS X comes with one pre-installed.

The computer being used to run OmegaT must have a Java Runtime Environment(JRE) installed on it—IT WILL NOT WORK WITHOUT ONE INSTALLED!

Java and OmegaT can be installed in any order. Configuration is not essential on UNIX based platforms, but it will make launching the application more user-friendly for those that require it.

If uncertain as to whether or not a JRE is installed, install OmegaT first and then test to see if it works (see OmegaT Installation). If nothing happens, or error messages are returned, the Java Runtime Environment is almost certainly not installed. In which case, please obtain one and install it first.

Please note that JRE is available in the form of both a standalone product (J2RE) or as part of the Java Software Developer's Kit(J2SDK).

This OmegaT release requires a JRE version of 1.4 or higher.

As far as the OmegaT project team is aware, OmegaT does NOT run on the following platforms:

  • Windows NT; the NT Java implementation contains bugs
  • Mac OS 9; the only Java Runtime Environment available for it (known as MRJ) is too old and does not offer the required functionality.

Java installation

Important Notes:

Sun regularly changes the names of its products—possibly to confuse its customers. You shouldn't care much, you just need a recent Java Runtime Environment. Version 1.4 or above, be it Java 2 version 1.4.2 or Java 5 version 1.5.0.

Other related (or not so related) forms of Java exist:

  • Java for embedded web browser applications (i.e. via plugins);
  • JavaScript scripting language for webpages (technically unrelated to Java).

Neither of these forms of Java will enable OmegaT to run.

Java on Windows

Check which version of JRE, if any, is installed with Start(button)/Windows key > Settings > Control Panel. If a JRE is already installed, a control panel applet labelled "Java" or similar will be present. Launch it and check whether it has the version 1.4 or higher.

Please note that as of this writing the current Java version available on Windows is in the 1.5 range (stable version). OmegaT is tested to work with this version.

1.4.x versions of OmegaT should work with any JRE version 1.4 upwards.

Users can download the Java Runtime Environment from:
Sun at http://java.sun.com/getjava/index.html

Installation instructions for JRE can be found on the website where it is downloaded from or accompanying the downloaded file or package once it has been opened or unarchived.

Java on UNIX (includes Linux & Solaris)

Type java -version at a command prompt in a console or terminal program. The version installed will be returned(if Java is installed as an executeable in your shell path).

Please note that as of this writing the current Java version available on Linux is in the 1.5 range (stable version). OmegaT is tested to work with this version.

1.4.x versions of OmegaT should work with any JRE version 1.4 upwards.

Users can download the Java Runtime Environment from:
Sun at http://java.sun.com/getjava/index.html
Blackdown (Linux) at http://www.blackdown.org/java-linux.html

Installation instructions for JRE can be found on the website where it is downloaded from or accompanying the downloaded file or package once it has been opened or unarchived.

Java on MacOSX

Mac users: On MacOSX, a JRE is pre-installed and configured. This is the only platform that does not require the user to install a JRE. If OmegaT does not work, make sure to run Software Update from the Apple Menu to get the latest JRE available for your version of OSX. OmegaT is known to work on OS X from OS X 10.2 to the most recent OS X 10.4.3 update.

Please note that as of this writing the current Java version available on MacOSX is Java 1.4.2. OmegaT is tested to work with this version.

Earlier versions of OmegaT (namely 1.0) are known to run on the MRJ for Macintosh OS 9. OmegaT version 1.4 or greater will not run on this platform.

To run an OmegaT 1.0 version, download the MRJ for Mac OS9 from the Apple website. OmegaT 1.0 is an obsolete version and this documentation makes no further reference to it.

OmegaT Installation

All Platforms

A generic OmegaT archive is available for installation. This version of OmegaT works on Linux, Solaris, MacOSX, and Windows (and possibly others). See System Requirements also.

Simply extract the contents of the .zip archive into the directory of your choice.

Windows

Windows users can download a Windows binary installer and just run it. In this case, the installer creates a shortcut to launch OmegaT from the Start > Programs menu.

MacOSX

OSX users can download a MacOSX package that automatically installs a MacOSX bundle into a specified directory. This package includes a docking icon and a double clickable application.

OmegaT Configuration

UNIX

This section is for UNIX users but MacOSX command line users can also benefit from it.

If installation on a UNIX based system is done by using the generic archive, and you are not very familiar with the platform, perform the following to make launching of OmegaT more user-friendly:

Save the following Bash shell script content as a text file:

#!/bin/bash
cd <OmegaT>
<java>/java -jar OmegaT.jar

where <OmegaT> is the directory containing the file OmegaT.jar and <java> is the directory containing the J2RE binary (the file named java). If the Java path variable is set, it is not necessary to specify this directory, and java -jar OmegaT.jar is sufficient to launch the program. Specifying the directory is necessary if the Java path variable is not set, and also if the system has more than one version of Java installed and a specific version needs to be used to launch OmegaT.

If Bash is not the default shell program change the first line of the script to reflect this (e.g. for Korn shell use #!/bin/kshin place of #!/bin/bash)

Save the script as OmegaT, omegat, or under whatever name is preferred. Then set the file permissions of the script to executable (e.g. via$ chmod u+x OmegaT in a shell, or if using Konqueror, context menu > Properties > Permissions > Exec). OmegaT can then be launched from the command line by typing the name given to the script. To launch from the desktop, create an icon and set it to execute the shell script file.

OmegaT Execution

Windows

If OmegaT has been installed with the help of the Windows installer (see above), it can be launched by clicking on the OmegaT shortcut from the Start > Programs > OmegaT menu. It is also possible to click on one of the files OmegaT.exe, OmegaT.bat, or OmegaT.jar in the file manager.

UNIX

Execute java -jar OmegaT.jar from the command line. Observe the exact type case of OmegaT—the capitals are important.

If Java has not been configured to detect the location of the OmegaT files, change directories to the directory containing the file OmegaT.jar.

If the Java path variable is not set, specify the complete path first. If neither of the above have been done, a routine to launch the application might look something like this for installation from the generic archive:

cd /home/ <user>/ <OmegaT install directory>
<JRE directory>/bin/java -jar OmegaT.jar

for user Joe Blow, archive installation directory OmegaT, and JRE directory

/usr/jre1.5

this would be

cd /home/joeblow/OmegaT
/usr/jre1.5/bin/java -jar OmegaT.jar

MacOSX

If the MacOSX bundle has been installed, simply click on the OmegaT icon located in the install directory. In the case where only the standard Java files have been downloaded and installed, simply click on the OmegaT.jar file in a Finder window.

OmegaT can also be launched from the Terminal (in a way similar to the command line under Linux). See above the UNIX configuration and execution sections for more information.

Changing the user interface language settings

The language used by OmegaT for the user interface and the user manual are set to a default value which depends on both your user environment and the available localizations. English will be used when there is no localization corresponding to your environment. For the list of available localizations, see below Available languages for OmegaT 1.6.

Any platform, from the command line

It is possible to override the default language used by OmegaT by entering the two letter language code, from the list given, to the -Duser.language java option (at the command line or in a shell script). This can be useful when using OmegaT in an environment where the default locale is not supported or is different than the one desired. For instance, the following command will start OmegaT in French regardless of whether other applications in the windowing environment are in French or not:

java -Duser.language=fr -jar OmegaT.jar

More specific locale settings are not supported for the localization of OmegaT at present (i.e. via the -Duser.country and -Duser.variant options passed to java).

Something else to note is the fact that for a language like Japanese it may be necessary to use a specific internationalized version of JRE from Sun to support character encodings properly. This is most likely in the case of Windows installation.

Any platform, with a python script

If Python is installed in your system, you can use a Python script providing a simple graphic language chooser. The script can be found at the OmegaT Yahoo! group site, [Files] section, [Macros and tools]. The script is OmegaTpy.zip contributed by Dmitri Gabinsky.

MacOSX, with the [System Preferences], [International] settings.

On MacOSX, changing the locale can be done from System Preferences in the Apple Menu by modifying the contents of the International preference pane. In the [Language] tab, drag the language you want OmegaT to run in to the top of the list. If the language is not on the list, click on [Edit List...] and add the language. Launch OmegaT with the modified setting and once OmegaT is launched, you can change the language order to make sure that applications started after OmegaT won't be affected by the language change.

Language Character Sets and Fonts

OmegaT provides for text display font settings.

In order to permit translation between two languages which employ different character sets, select a font which contains the character sets of both alphabets. Installation of additional fonts might be necessary in certain cases.

A different display font can be selected via the Display Font dialog. Open it via the Settings > Display Font... menu item. The font type and size can be changed from the dialog.

Note that the font selected applies only to the user interface display (i.e. the Editor and Match and Glossary Viewer windows. It has no effect on the fonts used within the source and target documents. The fonts available for selection are those which are available on your system.

Warning! It may take a noticeable amount of time for OmegaT to update the display after you have changed the font setting, especially when a big project is opened. Also, some fonts behave better for some language pairs than for others. Generally speaking, it is better to change of font before opening a project.
