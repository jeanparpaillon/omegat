Searches: Keyword and Exact

OmegaT provides a powerful range of search functions.

Open the search functions window by selecting Search Project... from the Edit menu or by pressing <Ctrl+F>. Enter a word or phrase to search for in the Search for text entry area.

Alternatively, select a word or phrase in the editing field (target text of the active segment) and press <Ctrl+F>. This way the word or phrase is automatically entered into the Search for box. Selecting is usually done by double clicking on a word or by dragging from the beginning to the end of a phrase with the mouse. On certain platforms this may not function in the same way, if at all.

Select the type of search using the radio buttons (keyword or exact search) and check boxes (search TMs/search files).

All search types given are case-insensitive. That is, a search for "abc" will find instances of "abc", "ABC", "Abc", etc.

Keyword Searches

Select Keyword search and enter any number of individual words, in any order, and OmegaT displays a list of all segments in which all the words occur. These searches are similar to a search "with all of the words" in an internet search engine, such as Google (i.e. AND logic).

Only whole words, not parts of words or incomplete words, can be found.

Only the project source files can be searched. Translation memories are not searched.

Exact Searches

An exact search looks for a phrase. Therefore, if several words are entered, they are only found when they occur in exactly the same sequence. An exact search is similar to an exact phrase search in an internet search engine.

Exact searches can look for search terms within:

  • source and target segments of the current project;

  • translation memories (files in /tm);

  • any file in a format supported by OmegaT (e.g. *.txt, *.html, *.xhtml, *.odt, *.sxw, *.properties), in selected directories and subdirectories (if required).

Wildcard characters (glob pattern characters) such as * and ? can be used in exact searches:

  • An asterisk * matches any character or number of characters. For instance, "b*d" matches instances of "bad", "bid", "bold" and "band";
  • A question mark ? matches any single character. "b?d" will match instances of "bad" and "bid".
