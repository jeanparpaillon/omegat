Glossary

OmegaT has an automatic glossary function. By placing an existing glossary, in the /glossary directory after creating a project, it will automatically be detected when a project is opened. Terms in the active segment which are found in the glossary file(s) are displayed in the Glossary pane of the Match and Glossary Viewer window. (See Translation Memory and Glossary Files for more information on glossary file formats.)

The glossary function only finds exact matches for the glossary entry (not inflected forms, etc.).

New terms can be added manually to the glossary file(s) during translation (for example in a text editor), but newly added terms will not be recognized until the project is reopened.
