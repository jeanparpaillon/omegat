ここでは、OmegaTの基本的な使い方を説明します。まずは、OmegaTの提供する機能を理解するため、このマニュアルをよく読んでください。

とりあえず：OmegaTはどのように働くか

OmegaTは、独立して移動できる2つのウィンドウを持っています。左側のウィンドウは、あなたの翻訳文を編集する"エディタウィンドウ"です。 右側のウィンドウは"立候補訳文訳文と語彙ウィンドウ"で、ファジー訳文(上側)と用語集(下側)があります。

OmegaTは、ソースドキュメントの文の区切りごとに動作し(これを文節と呼びます)、それを翻訳するエディタウィンドウに表示します。そして、あなたの翻訳文を翻訳メモリに追加します。翻訳を行った後(あるいは、あなたが必要なときに)、OmegaTは、ソースドキュメントの文節と翻訳メモリを使って、ターゲットドキュメントを再作成します。

OmegaTは、あなたの翻訳作業を"翻訳プロジェクト"として管理します。プロジェクトごとに、OmegaTは、ソースドキュメント・用語集・翻訳メモリをコピーしたフォルダ(ディレクトリ)のセットを作成します。さらに、OmegaTは、翻訳済みのドキュメントのフォルダも作成します。

OmegaTで、どのように翻訳するか

1.プロジェクトを新規作成して開く

メニューから、プロジェクト-> 新規作成を選択します。

プロジェクトファイルを保存したいフォルダを指定したら、"翻訳プロジェクト"名を入力します。この名前を、プロジェクトのフォルダ名として使います。すべてのプロジェクトファイルは、このフォルダに保存されます。

続いて、作成するプロジェクトフォルダを設定できるダイアログが表示されます。これは次図のように表示されます。

サブフォルダの場所は、そのままで構いません(OmegaTは、これを "ディレクトリ"と呼びます)。ただし、ソースとターゲット言語のコードを正しく選択してください。 言語コードまたは言語-国別コードをドロップダウンリストで選択するか、キーボードから入力します。このとき、"target"フォルダの場所を覚えておいてください。ここにターゲットドキュメントが作成されます。

もしも、OmegaTの文節を段落の代わりに文で区切りたいなら、セグメントルールを調べるといいでしょう。

プロジェクトの設定でOKをクリックすると、OmegaTはソースドキュメントをインポートするよう指示します。ファイルは個別にインポートできますが、ディレクトリツリー全体をインポートするようにしましょう(サブフォルダの全ファイルが対象になります)。もしも、不正なドキュメントやドキュメントの数が多すぎてインポートがうまくいかない場合は、プロジェクトの"source"フォルダから、そのファイルを削除してください。

OmegaTが翻訳できるのは、OpenDocumentファイルおよびOpenOffice.org ドキュメント, HTML, プレーンテキスト、Javaファイルです。他の種類のファイルは、OmegaTは対応していません。

表示-> プロジェクトファイル...をオンにして、プロジェクト-> 再読込を選択すると、翻訳できるファイルがリストに表示されます。 OmegaTは、デフォルトで、リストの最初のファイルを読み込みます。OmegaTは、デフォルトで、リストの最初のファイルを読み込みます。

2.文節を１つずつ翻訳する

選択中の文節において, OmegaTは、ソーステキストを緑色の背景色で表示します。そして、 <文節 0001> と </文節>という2つのタグの間に、ソーステキストのコピーを表示します。2つのタグの間に、翻訳文を上書きしてください。

fuzzyマッチがある場合、そのテキストをエディタウィンドウにfuzzyマッチしたテキストを次のようにして挿入できます。

  • Ctrl+1, Ctrl+2 などを使って、正しいfuzzyマッチを選択します。

3.訳文ファイルを生成する

翻訳を行った後(あるいは、あなたが必要なときに)、OmegaTは、ソースドキュメントの文節と翻訳メモリを使って、ターゲットドキュメントを再作成します。

訳文ファイルを生成するには, メニューからプロジェクト-> 訳文ファイル生成を選択します。 OmegaTは、どれだけ翻訳しているかに関係なく、プロジェクトの"source"フォルダに、正しいドキュメントのターゲット版を作成します。

"ターゲットファイルを作成"を呼び出せば、ターゲットドキュメントの状態をいつでも確認できます。あなたは、翻訳プロジェクトにある訳文をいつでも変更できるし、好きなだけターゲットドキュメントを再作成できます。

翻訳されたファイルは、プロジェクトの"target"フォルダにあります。

タグ付けテキストの対処法

OpenDocumentファイルや,OpenOffice.orgドキュメント、HTMLファイルでは、書式文中の単語にそれぞれ独自の書式を使っています。たとえば、文全体はイタリックで表示しておいて、文中の単語は太字で表示するという具合です。

OmegaTは、これらの書式を特別なタグを使って表し、それをテキストの翻訳版で使います。OmegaTユーザーズマニュアルのtag editingの拡張を見てください。

OmegaTは、タグの翻訳間違いを自動的に見つけません。ターゲットドキュメントを顧客に渡す前に、タグのエラーがないか、メニューからツール->タグ検証を選択して確認してください。さらに、ドキュメントを開くか、ビューアで確認すれば間違いありません。
